# 一、本章拟定节结构（对应 v5.0：动机→反面→历史→理论锚定→陷阱→练习）

## §arch.1 动机：四足控制为什么「难且特别」
- `insight`（作者洞见·主线）：**足式运动控制本质是一个优化/规划问题**——在「路径最短/速度最快/能耗最小」等指标下、满足「加速度/速度上限、不可涉足空间」等约束求最优控制量；多数情况无法完美建模或非线性难解，故牺牲部分解空间换次优可行解（[笔记] 行 25）。
- `insight`（作者洞见·主线）：**车辆/无人机是「连续单模型」，四足是「多模型须切换」**——后二者运动模式与约束几乎不变可用统一模型；四足存在站立/支撑/摆动/飞行等多种运动模型，必须做**模式切换**（[笔记] 行 29）。这条直接导出后文 FSM（§arch.4.5）的必要性。
- `insight`：足式相对轮/履的两大本质优势——(i) **非连续支撑**可越离散支撑点通过非结构化地形（爬楼梯例，[笔记] 行 15、[于宪元] 行 244）；(ii) **本体与腿运动解耦**，类主动悬挂避振（[笔记] 行 15、[于宪元] 行 253）。代价：结构复杂、能耗高、续航短（[笔记] 行 21）。

## §arch.2 反面：朴素思路为什么不行
- **反面 1**：用「固定运动模式的连杆机构」（切比雪夫机构等）→ 机身受扰无法响应、不能适应地形变化；20 世纪 50 年代已认识到「真正的足式机器人必须有反馈系统」（[于宪元] 行 269）。`pitfall`/历史教训。
- **反面 2**：纯**位置控制**（规划足底轨迹→逆运动学→发关节位置）→ 与 19 世纪连杆机器人无本质区别；电机内部位置环是「黑箱」，与环境有力作用时行为不可控；位置环须响应时间→控制带宽低（[于宪元] 行 278–284）。`pitfall`。
- **反面 3**：只用瞬时/无预测控制器（如纯 WBC 或瞬时 QP）→ 在飞行相/长时间欠驱步态下，支撑相后半程不为飞行相做准备、飞行相后半程不为落地冲击做准备（[于宪元] 行 327；[笔记] 行 79）。`pitfall`，导出 MPC 必要性（§arch.4.3）。
- **反面 4**：早期 RL 模型参数少、手工设计、硬件微调 → 表征能力受限（[笔记] 行 87）。

## §arch.3 历史与平台谱：从连杆到 MPC+WBC（6 里程碑 + 平台史）
- **里程碑链**（[于宪元] 行 261–338，逐个 insight/`paper`）：
  1. 固定连杆机构（迈布里奇 1878 抓拍腾空相；切比雪夫连杆机构）→ 无反馈。
  2. 可自由设计足底轨迹的**位置控制** → 仍无力控本质。
  3. **由训练有素的人类驾驶**（GE 四足卡车，液压+力反馈把手）→ 闭环但靠人。
  4. **Raibert 三通道解耦控制**（高度/姿态/速度分通道；支撑相借力控高度姿态，飞行相控落足点调速度）+ **SLIP 模型**（弹簧负载倒立摆）→ 证明「精准动力学非绝对必要」（[于宪元] 行 295–312）。`\cite{raibert1986legged}`。
  5. **Pratt 虚拟模型控制 VMC**（虚拟弹簧/阻尼/质量产生等效关节力矩，计算量小、可自由设虚拟组件）（[于宪元] 行 314–323）。`\cite{pratt2001virtual}`。
  6. **WBC**（任务分解+优先级，零空间保证低优先级不扰高优先级；本书 WBC 至少实现浮动基逆动力学）（[于宪元] 行 325）。**MPC**（滚动时域优化跟踪期望轨迹，弥补 WBC 无预测；但在线大规模优化算力高、带宽受限）（[于宪元] 行 327–336）。`\cite{dicarlo2018mit}`。
- **总结演进**（[于宪元] 行 338）：从无到有、从简化模型到全动力学、从瞬时控制到预测控制——形成「以 MPC 求足底反力 + WBC 转关节力矩 + VMC/SLIP 理念」的现代局面。
- **平台谱**（[宇树书] 行 186–194，`practice`/表）：波士顿动力（Atlas/Spot/Stretch，源出 MIT Leg 实验室 Raibert）、MIT Biomimetic（Cheetah 3 / Mini Cheetah，首个后空翻四足，开源软硬件）、ETH RSL（ANYmal）、IIT DLS（液压 HyQ）、UPenn GRASP/Ghost Robotics、宇树（Laikago 首台公开零售→Aliengo/A1/Go1）。
- **硬件/模拟器/算法三线演进**（[笔记] 行 33–91，`practice`/凝练）：驱动三路（液压/电机/电液混合）；模拟器从弹簧阻尼惩罚法→GPU 并行（Gazebo/MuJoCo/Isaac Lab，每秒数十万状态转换支撑 RL）；算法两流派（优化 vs RL）。

## §arch.4 理论锚定：四足动态运动控制的分层架构（本章核心）
> 用于宪元三问作主轴，把个人笔记「运动控制框架」展开为分层数据流，并为每层 `\cref` 到专章。
- §arch.4.1 **架构总览**：上层规划指令（外部路径规划器/遥控器速度指令 + 步态调度器步态指令）→ 状态估计 → 质心/足底规划 → MPC（支撑相求足底反力）→ WBC（多刚体逆动力学转关节力矩）→ 电机（[笔记] 行 95；[于宪元] 行 378）。实线=控制信号流，虚线=状态信号流。
- §arch.4.2 **「在哪儿」= 状态估计**（[于宪元] 行 344）：躯干旋转由 IMU 经坐标变换得姿态/角速度；躯干平动用 KF 足底里程计（支撑足不打滑假设，建本体系↔世界系足底速度映射求质心位置/速度）。`\cref{ch:state_estimation}`。
- §arch.4.3 **「去哪儿」= 规划**（[于宪元] 行 346）：质心轨迹规划器（解算期望质心轨迹）+ 足底轨迹规划器（按实际/期望水平速度偏差选落脚点，在离地点与落脚点间规划摆线轨迹）。`\cref{ch:gait}`。
- §arch.4.4 **「怎么去」= MPC + WBC**（[于宪元] 行 348）：MPC 把整机简化为单刚体→状态/预测方程→优化求足底反力（`\cref{ch:srbd}`/`\cref{ch:convex_mpc}`）；WBC 用多刚体动力学，任务按重要程度分 4 优先级，雅可比零空间保证优先级（`\cref{ch:wbc}`）；最后多刚体动力学求关节扭矩（`\cref{ch:rbd}`）。**频率解耦**：MPC 周期 ΔT=0.01 s，其余 Δt=0.002 s（[于宪元] 行 378）。
- §arch.4.5 **相位/行为组织 = FSM**：多运动模型须切换（呼应 §arch.1 洞见）；FSM 管理起立/下蹲/行走/平衡站立等行为，含切换关系约束 + 忙碌时间 + 危险检测→阻尼保护（[于宪元] 行 389–417）。

## §arch.5 陷阱清单（`pitfall` 集）
- 把瞬时控制当全部 → 飞行相崩（反面 3）；位置控制黑箱 → 力交互不可控（反面 2）；忽略模式切换 → 单一模型跑不了 trot/jump；状态机非法切换（下蹲后直接行走）/忙碌时间未等待 → 危险（[于宪元] 行 391）。

# 二、逐条收录的关键公式（公式少、定位性为主、出处明确）

## 式 arch.1 —— FSM 阻尼保护控制律 ★（本章唯一一手公式，安全机制）
出处：[于宪元] §2.1 式 (2.1)，源行 406。`\cite{yu2021quadruped}`
$$
\tau_{\mathrm{motor}} = -\,k_{\mathrm{damping}}\,\cdot v_{\mathrm{motor}}
$$
- **来由**：危险触发后，12 个电机各自独立施加与转速反向、正比于转速的力矩（纯速度阻尼），使机器人快速平稳静止（[于宪元] 行 404、417）。$k_{\mathrm{damping}}$ 阻尼参数（调试定），$v_{\mathrm{motor}}$ 电机转速；转速越大对外阻尼力矩越大。
- **触发表**（`practice`/表，源行 420–423）：俯仰+横滚之和 > 35°；机身水平速度 > 5.5 m/s；足底速度 > 5.5 m/s；关节速度 > 18°/s；关节跟踪误差 > 20°；足底跟踪误差 > 0.2 m。
- **主线锚点**：这是「多模型切换」哲学的安全落点——任何受控状态检测到危险即自动调度到阻尼保护态（[于宪元] 行 393）。OCR 见 §四第 1 条。
- **\cref**：完整 PD+前馈关节控制律见 `ch:prac`（式 prac.1）；本式为其退化（$k_p=0,\tau_{\mathrm{ff}}=0,p_{\mathrm{des}}=\omega_{\mathrm{des}}=0$）。

## 式 arch.2 —— 控制频率/周期（架构定位，凝练登记）
出处：[于宪元] §2 总体介绍，源行 378。
$$
\Delta T = 0.01\ \text{s}\ (\text{MPC}),\qquad \Delta t = 0.002\ \text{s}\ (\text{其余模块})
$$
- **来由**：MPC 在线解大规模优化、算力重、频率低（100 Hz）；状态估计/WBC/规划等高频（500 Hz）。解释级联架构的**频率解耦**（MPC 慢环 + WBC 快环）。`note`：原文「除 MPC 频率为 □ 外其他为 □」处 MinerU 漏字，据 ΔT/Δt 反推为 100 Hz / 500 Hz（见 §四第 2 条）。
- **\cref**：`ch:convex_mpc`（MPC 慢环）、`ch:wbc`（WBC 快环）。

## 式 arch.3 —— 足底反力 → 关节力矩的定位式（虚功，本章只「点名」）
出处：综合 [笔记] 行 95 架构 + [于宪元] 行 348（MPC 求足底反力、WBC/多刚体转扭矩）。详证留 `\cref{ch:wbc}`/`ch:prac`。
$$
\boldsymbol\tau = \boldsymbol J^{\mathrm T}\boldsymbol f
$$
- **来由（本章仅一句话定位）**：MPC/QP 在「整机/世界系」求出的足底反力 $\boldsymbol f$，最终须由各腿雅可比转置映射回关节力矩 $\boldsymbol\tau$ 发给电机——这是「足端力世界→关节空间」的最后一公里。完整虚功推导与作用反作用符号见 `ch:prac` 式 prac.2。
- **全局约定 `note`**：本书右扰动主线、旋转记 $\boldsymbol R_{sb}$（机身→世界），世界系足底力 $\boldsymbol f_{is}$ 转关节力矩为 $\boldsymbol\tau_i=-\boldsymbol J_i^{\mathrm T}\boldsymbol R_{sb}^{\mathrm T}\boldsymbol f_{is}$（与宇树书/于宪元一致，无需换算）。

## 式 arch.4 —— SLIP 概念模型（历史里程碑，示意式）
出处：[于宪元] 行 312（Raibert 把单/双/四足均抽象为弹簧负载倒立摆）。`\cite{raibert1986legged}`
$$
m\,\ddot{\boldsymbol r} = \boldsymbol F_{\mathrm{leg}} + m\boldsymbol g,\qquad
\boldsymbol F_{\mathrm{leg}} = k_{\mathrm{leg}}\,(\ell_0-\ell)\,\hat{\boldsymbol n}
$$
- **来由（仅作历史示意，不展开）**：SLIP = 点质量 $m$ + 无质量弹性腿（刚度 $k_{\mathrm{leg}}$、自然长 $\ell_0$、当前长 $\ell$、沿腿方向 $\hat{\boldsymbol n}$）。Raibert 用以论证「简单模型即可在不同地形运动、精准动力学非必要」（[于宪元] 行 312）。
- **`note`（OCR/订正说明）**：原文未给 SLIP 显式方程，仅文字描述「弹簧负载倒立摆」；此式为据 SLIP 标准定义（`\cite{raibert1986legged}`）补的**示意式**，写正文时以 `remark`/历史盒呈现并注明「标准 SLIP 形式，非原文公式」，**不得当作于宪元原文公式引用**。

> **公式收录小结**：一手安全式 1 条（arch.1）+ 架构定位式 2 条（arch.2 频率、arch.3 J^T 点名）+ 历史示意式 1 条（arch.4 SLIP，补充式标注）= **4 条**。本章为导论，VMC/WBC/MPC 的实质公式全部 `\cref` 专章，**不在本章推导**。

# 三、个人笔记中的「作者思路 / 洞见」点（本章主线 + insight 盒）

> 这些是教科书不直述的作者视角，作主线骨架，逐条标「作者洞见」做 `insight`。

1. **〔主线·控制=优化/规划〕** 足式运动控制本质是优化/规划问题：在指标（最短/最快/最省能）下满足约束求最优控制量；无法完美建模或非线性难解时，牺牲解空间换次优可行解（[笔记] 行 25）。**全章总纲，开篇 insight。**
2. **〔主线·连续单模型 vs 多模型切换〕** 车辆/无人机运动连续、约束不变可用统一模型；四足存在多种运动模型须切换（[笔记] 行 29）。**直接导出 FSM 必要性，串 §arch.1↔§arch.4.5。**
3. **〔解的去向〕** 上层求出的解是轨迹，发给底层跟踪器跟踪；观测器（IMU/轮速计）实时估计当前状态（位置/速度/加速度）（[笔记] 行 27）。**导出「估计→规划→控制」闭环数据流。**
4. **〔足式两大本质优势〕** 非连续支撑通过非结构化地形 + 本体与腿运动解耦（类主动悬挂）（[笔记] 行 15）。`insight`，呼应于宪元行 244/253。
5. **〔硬件是地基〕** 「足式机器人的发展离不开硬件的发展，就跟 AI 离不开芯片」——现代高功率密度可控扭矩电机使低成本高性能四足成为可能（[笔记] 行 35、51）。`insight`/历史盒。
6. **〔模拟器使能 RL〕** GPU 并行模拟器（Isaac Lab 每秒数十万状态转换）保证 RL 大规模采样可行；惩罚法弹簧阻尼难模刚性接触（[笔记] 行 65、67）。`insight`。
7. **〔优化 vs RL 两流派〕** 优化法建模动力学、最小化代价求未来最优控制量（足端力）；RL 基于奖励探索、最大化回报求策略；二者本质都是「基于某函数算成本/奖励求最优运动策略」（[笔记] 行 75–91）。`insight`，本章末「另一条腿」点到为止。
8. **〔接触-规划分离〕** MPC 范式采用「接触与运动规划分离」降复杂度，结合 WBC 取得好效果；但优化法难扩展到接触不确定性/感知融合，故 RL 兴起（[笔记] 行 81–83）。`insight`，串 MPC 与 RL。
9. **〔于宪元三问助记〕** 「机器人在哪儿（状态估计）/去哪儿（规划）/怎么去（MPC+WBC）」（[于宪元] 行 343）。**作分层架构的助记主轴，贯穿 §arch.4。**
10. **〔浮动基欠驱本质〕** 足式机器人是六自由度欠驱浮动基座，且该基座正是主要被控对象；只能靠关节扭矩调与环境的交互力间接维持基座姿态/速度（[于宪元] 行 265）。`insight`，解释「为何要力控+足底反力」。
11. **〔地面反力是唯一外力〕** 除重力外，地面反力是浮动基机器人受到的唯一外力——故控制系统注重与环境的作用力而非位置（[于宪元] 行 280）。`insight`，导出力控范式。
12. **〔集各家之长〕** 本书控制系统：MPC 求足底反力 + WBC 算关节空间期望并经多刚体动力学求扭矩 + 借鉴 VMC/SLIP（[于宪元] 行 338）。**本部各章如何拼成一张图的总声明。**
