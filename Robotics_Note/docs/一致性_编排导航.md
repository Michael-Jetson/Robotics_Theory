## 1. 难度分层标注：**全书 5 种 glyph 并存**（贯穿性问题，最该统一）

任务点名"同一内容在导航表与正文标题的星级/选读标记是否一致（如 STEAM、GVI、SE₂(3)、InEKF 都应标进阶/选读且各处口径统一）"。逐章 grep 选读/进阶标记后，**全书的'进阶/选读'标题用了五种不同写法**：

| glyph | 出现处 |
|---|---|
| `$*$`（数学星号） | lie_theory ×4（插值/Sim3/对称/最小多项式）、point_cloud（D2D-NDT 1195）、matrix_calculus（"选读 $*$"×3） |
| `$\star$`（五角星） | slam_state_estimation（STEAM）、kalman_eskf ×2、nonlinear_optimization（GVI 父节）、point_cloud（de Ruiter-Forbes 587） |
| `$\dagger$`（剑标） | nonlinear_optimization ×2（GVI 最深两子节 1313/1374） |
| `$\bigstar$`（大星） | matrix_calculus（Van Loan 545） |
| 中文"进阶" | imu_model（1103） |

**同章内自相矛盾的两处（最该先修）**：
- **point_cloud_processing**：`$\star$`（587 de Ruiter-Forbes）与 `$*$`（1195 D2D-NDT）混用；而章首导航推荐路径（67 行）只写"带 $*$ 的 D2D-NDT…为研究级选读"，**完全没解释 `$\star$` 是什么**——读者看到 587 行的 `$\star$` 无从对照。
- **matrix_calculus**：`(选读 $*$)`（381/427/643）与 `$\bigstar$`（545 Van Loan）混用于同一附录。Van Loan 注释自述是"STEAM/IMU 离散化的标准引擎"，疑似想用 `$\bigstar$` 表示"更重要"，但既无"选读"字样也未在任何图例说明，口径与同附录其余三节割裂。

**SE₂(3) 口径不一（任务点名项）**：`sec:se23`（lie_theory 977 行）标题**无任何星标**，但同章导航推荐路径（65 行）明确把"$\mathrm{SE}_2(3)$"列进"带 $*$ 的…为进阶/选读"一串里。即"导航说它选读、正文标题却不标"。对比同段并列的插值（663 有 `$*$`）、Sim(3)（921 有 `$*$`）、对称（1066 有 `$*$`）——唯独 SE₂(3) 漏了星号。**建议**：把 977 行标题改 `\section{$*$扩展位姿群 $\mathrm{SE}_2(3)$…}`（与同章其余进阶节统一）。

**InEKF / 互补滤波（任务点名项）口径核查 = 通过**：kalman_eskf 的 InEKF（`paper:inekf`）与互补滤波两段都在 §7 ESKF（骨干）内、**导航与总表都把它们归骨干、正文也不标星**——三处口径一致，无矛盾（不必加星）。

**建议**（分层标注，全书级）：统一一套图例，例如 `$\star$ = 进阶/选读`、`$\dagger$ = 最深旁支（可完全跳过）`，并在每章导航推荐路径处用同一 glyph；至少先消除 point_cloud 与 matrix_calculus 两处**同章混用**。`$\dagger$`（nonlinear_optimization 1313/1374，planning_intro 541 亦有）若确为"比 $\star$ 更深一档"的有意约定，应在导航图例补一句定义（现 70 行只说"带 $\star$ 的小节为进阶"，读者对 `$\dagger$` 无键可查）。
