# 0 章首与传感器背景（与点云"从何而来"相关的部分）（源 行 1–37）

> 说明：点云处理章一般需要交代"点云的物理来源 / 采集机制 / 稀疏-稠密-畸变特性"。本源 §8.1 提供了这部分背景，与点云数据结构的设计动机直接相关，故保留要点。

**LiDAR 工作原理（源行 5、17）**：LiDAR 主动发射激光脉冲，测量脉冲被表面反射回探测器的**时间延迟**，直接测距，从而感知周围环境结构。最基本原理为**飞行时间（Time-of-Flight, TOF）**：用激光脉冲、依"发射光返回探测器所用时间"推断距离。TOF 可得高分辨率测量，但对外部光敏感，会降低信噪比（SNR）[598]，进而降低测量精度与频率。除 TOF 外，源于雷达的**调幅连续波（AMCW）**与**调频连续波（FMCW）**也被用于 LiDAR。

**LiDAR 传感器分类（源行 19，依据 [935]）**：① 机械式 LiDAR；② 扫描式固态 LiDAR；③ 闪光 LiDAR；④ 宏观扫描 Risley 棱镜。本章重点考察 **2D/3D 机械式 LiDAR** 与 **宏观扫描 LiDAR**。

**点云的稀疏-稠密不均匀性（点云处理的关键先验）**：
- **图 8.2（源行 29）技术细节**：扫描来自 64 波束 Hesai QT64 扫描仪，垂直 FOV 104°。**注意可见的单条扫描线，以及点云在靠近设备处很稠密、在远处稀疏得多。** ——这条"近密远疏 + 沿扫描线分层"的结构是后续体素化、降采样、最近邻搜索、特征提取都必须考虑的点云先验。
- **3D 机械式 LiDAR（源行 23–31）**：多个激光发射器装在单一旋转机构上，随整机构在方位轴（azimuthal）旋转 360° 时，各自捕获指向不同仰角（elevation）的距离测量（图 8.1b）。点云可高度详尽刻画物体与周围环境。开创性传感器为 **Velodyne HDL-64E**（DARPA Grand Challenges 催生）。
- **2D 机械式 LiDAR（源行 21）**：旋转镜引导单束激光测距，编码盘测角，距离+角度配对得 2D 剖面；本质只能在单个 2D 平面内扫描。

**每点的附加通道（点云的字段）（源行 37）**：以上传感器都产生一组带**强度（intensity，亦称 remission）值**的单独距离测量；带各波束角度的距离测量可转为 2D 或 3D 点云。新进展引入超出默认测距的额外测量能力：
- **FMCW LiDAR**：连续投射变频光，用频移测被击中物体的**相对速度**（类似 FMCW 雷达），动态场景有用 [1201]，但更复杂昂贵。
- **闪光 LiDAR**：能提供类似相机的"环境通道（ambient channels）"，类似光度（photometric）测量。

> **对点云处理章的提示**：因此一个"点"在不同 LiDAR 下可能携带的字段为 $\{x,y,z\}$（基本）、$+\,\text{intensity}$（几乎所有）、$+\,\text{相对速度}$（FMCW）、$+\,\text{ambient/光度}$（flash）、以及隐含的 $\{\text{ring/波束号},\text{时间戳}\}$（多波束、用于去畸变）。本源未把这些字段统一列成"点的数据结构"，综合时可据此整理。

**图 8.1（常见 LiDAR 类型及波束模式，源行 13）技术细节**：(a) 2D LiDAR——旋转发射镜（黄）+ 编码盘（蓝）测旋转角；(b) 机械多波束——多发射器（黄）发射、多探测器（蓝）接收，整头旋转得 360° 水平 FOV；(c) 宏观操纵 Risley 棱镜——螺旋扫描，只在透镜窗口方向（青）测量，螺旋模式随时间产生更稠密点云。

# 8.2 LiDAR 里程计中的点云配准（与点云处理强相关的总入口）（源 行 39–47）

**scan（扫描）的定义（源行 41）**：LiDAR 传感器采集的一次单独扫掠（sweep）或循环（cycle）；通常代表一次完整旋转或一次完整扫掠，提供某一时刻周围环境的上下文快照；通常带时间戳，可排序并作为序列观测处理。**一帧 scan 实际就是一组点 / 一个点云。**

**核心技术 = scan registration（扫描配准）/ scan matching（扫描匹配）（源行 43）**：精细对齐一对 scan，估计 scan 间精确相对变换。文献中点云配准的重要算法：
- **Horn's method [477]**：提供**闭式解（closed-form solution）**，但需**预先确定的对应关系**，限制其真实场景适用性。
- **ICP（Iterative Closest Point）[79, 959]**：**不需要先验对应关系**，因此成为确定点云间相对变换的更流行、更基础的技术。

> **抽取者注（给综合）**：源文在这里明确点出了点云配准两大经典路线的本质差异——**已知对应 → 闭式解（Horn / SVD 类）** vs **未知对应 → 迭代（ICP）**。这正是点云处理章讲 ICP 前应铺垫的对比，但**源文未给 Horn's method 的任何公式**（无 SVD、无四元数闭式解）。

**2D→3D 点云累积的早期做法（源行 45–47）**（与"如何得到稠密 3D 点云"相关）：在 2D 扫描技术基础上，主动以**点头（nodding）[433]** 或**旋转（rotating）[100]** 方式移动传感器，或被动利用人体携带者 [102] / 车辆 [100] 的运动来累积更稠密 3D 点云；这些 3D 点云使 3D scan matching 成为可能，但因数据量增大带来显著计算挑战。**LOAM [1269]** 应对挑战、展示了**实时** scan matching，成为后续方法基础。

# 8.2.1 Scan Registration 的基础（点云配准核心）（源 行 49–104；图 8.3、图 8.4）

Scan registration 是 LiDAR 里程计与建图系统的基础组件，涉及对齐两帧 scan。目标：找到变换——旋转 $\mathbf R\in SO(3)$ 与平移 $\mathbf t\in\mathbb R^3$——使其最好地把一帧 scan（可能是当前帧）对齐到另一帧（如上一帧或局部地图），同时给出两帧采集位置间的相对位置。已有大量技术以高精度、高鲁棒、低算力完成配准。

## 8.2.1.0 点云的形式化定义（本源给出的唯一"点云表示"公式）（源 行 53）

如第 5 章所述，**点云**定义为三维坐标系中的点集，数学表示为

$$
P=\{p_i\in\mathbb{R}^3 \mid i=1,2,\dots,N\},
$$

其中每个

$$
p_i=(x_i,y_i,z_i)
$$

表示一个点的 3D 坐标。

> 这是本源关于"点云表示"的**全部**形式化内容。其余表示（range image、voxel、surfel、BEV）只在后文以文字点名（见本抽取 §"其它点云表示"汇总），技术细节均指向 Chapter 5。

## 8.2.1.1 ICP 的统一目标函数（式 8.1 / 8.2）（源 行 53–63）

对 scan registration，ICP 算法 [79] 最小化两个点云 $P$、$Q$ 间的总配准误差。记 $P$ = **source（源）**、$Q$ = **target（目标）**。ICP 在优化迭代 $k$ 中**迭代地**确定变换 $\mathbf R^k,\mathbf t^k$，最小化由不同距离度量 $d$ 衡量的总配准误差：

> **【式 8.1】ICP 目标函数（源小节 8.2.1）**
> $$\mathbf{R}^{k}, \mathbf{t}^{k} = \underset{\mathbf{R}, \mathbf{t}}{\operatorname{arg\,min}} \sum_{(\mathbf{p}, \mathbf{q}) \in \mathsf{C}} d\big(\mathbf{p}_{i},\ \mathbf{R}\mathbf{q}_{i} + \mathbf{t}\big), \tag{8.1}$$

其中 source 点云 $P$ 与 target 点云 $Q$ 间的对应集合 $\mathsf C$ 为：

> **【式 8.2】对应集合的一般定义（源小节 8.2.1）**
> $$C = \{ (\boldsymbol{p}, \boldsymbol{q}) \mid \boldsymbol{p} \in P,\ \boldsymbol{q} \in Q \}. \tag{8.2}$$

在 ICP 中，source 与 target 间的变换通过迭代实现：**每次迭代 $k$ 都基于上一次迭代 $k-1$ 的变换 $(\mathbf R^{k-1},\mathbf t^{k-1})$ 重新计算一组新的对应 $\mathsf C$。** （这就是 ICP 的"迭代最近点"双步循环：① 用当前变换重算对应；② 固定对应解最优变换。源文只给出这一**框架性**描述，**未展开第②步如何解、也未证明整个迭代收敛**。）

**要最小化式 (8.1)，必须指定两个方面（源行 65–73）：**
1. **用什么几何关系定义距离度量 $d(\cdot)$？** 目标是把两点云对得尽可能紧（tight），而这种"紧"在未定义距离度量前无法确定。
2. **如何确定用于最小化的对应集合 $\mathsf C$？** 即为每个 source 点 $p\in P$ 找对应 target 点 $q\in Q$。

**图 8.3（ICP 中典型距离度量，源行 71）技术细节**：
- **(a) 点-点距离（point-to-point）**：两点间欧氏距离，直截了当。
- **(b)、(c) 点-高层特征距离**（线 line 或面 plane）：到"用 target 点重建出的线/面"的**最短距离**。

## 8.2.1.1（续）配准残差中的距离度量——ICP 的三种残差类型（源 行 77–88）

第一个问题涉及选用哪些几何元素定义残差，可能超越简单点-点。通常**点、线、面**是定义式 (8.1) 距离的最常见几何元素（图 8.3 总结）。逐项全量保留：

- **点-点 ICP（Point-to-point ICP）**：最基础，最小化两点云间对应点的**欧氏距离**。开创性工作——Zhang [1281] 与 Besl & McKay [79]——把形状（曲线/曲面）匹配表述为**点匹配**问题，把形状表示为点集。点-点代价直接简单，但**对噪声、外点（outliers）、点的稀疏性敏感**。
- **点-线距离（point-to-line）**：度量一个点云中的点与另一个点云中（由连接点形成的）**线**之间的距离；在带线性特征的结构化环境中能给更好结果。
- **点-面距离（point-to-plane）**：度量一个点云中的点与另一个点云中的**面（局部曲面）**之间的距离；对噪声更鲁棒，在带平面表面的环境中可达更高精度。

**其它 ICP 变体的距离度量（源行 83–88）**：
- **多距离度量（multi-distance metrics）** [839]
- **连续时间表述（continuous-time formulation）** [256]
- **自适应阈值（adaptive thresholds）** [1133]
- 另一些方法 [985, 591] 选择**比较局部邻域的概率分布之差**，而非用欧氏距离。（按引文，[985] 即 Generalized-ICP 谱系、[591] 为相关分布法；但源文**未点 GICP 之名、未给任何分布加权残差公式**。）
- **正态分布变换（Normal Distributions Transform, NDT）[82]**：另一种著名的**基于分布的匹配**。NDT 把输入点云分成一组**体素（voxels）**，对每个体素内的点拟合一个**正态分布**（细节见 Chapter 5.3.2.2）。它**不付出确定最近邻关联的代价**，而是利用体素化进行**分布-分布匹配（distribution-to-distribution matching）**；此过程能利用更平滑、更鲁棒的配准代价曲面，在复杂环境中尤为有利。

> **综合 agent 注（point-to-plane / GICP / NDT 推导缺失）**：本源对这三类的描述到此为止，**全是文字**。
> - **point-to-plane** 的标准残差 $r_i=\mathbf n_i^\top(\mathbf R\mathbf p_i+\mathbf t-\mathbf q_i)$、其线性化为 $6\times6$ 法方程的推导——**本源无**。
> - **GICP** 的 plane-to-plane / 协方差加权马氏残差 $\sum_i d_i^\top(C_i^Q+\mathbf R C_i^P\mathbf R^\top)^{-1}d_i$——**本源无、连名字都没有**。
> - **NDT** 的 score 函数 $\sum_i \exp\!\big(-\tfrac12(\mathbf x_i-\boldsymbol\mu_k)^\top\Sigma_k^{-1}(\mathbf x_i-\boldsymbol\mu_k)\big)$、其牛顿优化的梯度/海森——**本源无**。
> 综合时这三块务必另寻源。

## 8.2.1.2 数据关联 / 对应搜索（最近邻 / KD-tree）（源 行 89–102）

ICP 的第二个核心设计决策是 source $P$ 与 target $Q$ 之间的**数据关联（data association）/ 对应搜索（correspondence search）**。

最基本形式：用迭代更新的变换 $(\mathbf R^{k-1},\mathbf t^{k-1})$，几何地为点 $p\in P$ 在 target $Q$ 中找**最近邻**：

> **【式 8.3】基于最近邻的对应集合（源小节 8.2.1）**
> $$C = \Big\{ (p, q)\ \Big|\ p \in P,\ q = \underset{q' \in Q}{\arg\min}\ \big\|p - (R^{k-1}q' + t^{k-1})\big\|_2 \Big\} \tag{8.3}$$

**但在含数千点的大点云上，这通常是昂贵操作。** （这正是点云处理章引入空间数据结构/KD-tree 的动机；源文在此点明动机，但**未给 KD-tree 算法**。）

**为使 LiDAR 里程计可实时运行，减少对应搜索时间的两大策略（源行 98）：**
1. **缩减对应候选集合**（reducing the set of potential candidates for a correspondence）；
2. **采用不同于"精确距离最近邻搜索"的搜索策略**，以更快找到潜在候选。

**策略一的具体做法（源行 100）**——通过构建"候选集已缩减的地图"减少潜在对应，即取满足某些几何准则的点 $P'\subset Q$、$Q'\subset Q$（源文如此，两处皆写 $\subset Q$）。所用准则（全量保留）：
- 确定位于**边（edges）或面（surfaces）**上的点 [1269, 839]（即 LOAM 类特征筛选，见 §8.2.2.2）；
- **移除描述性较弱的、对应于地平面（ground plane）的点** [998]；
- 对 target scan 做**降采样 / 下采样（downsampling）** [1133, 256]。
- **潜在缺点**：这些策略确实加速对应搜索，但**有"从 target $Q$ 中移除真正对应"的风险**。

> 这是本源关于「滤波/降采样」的**全部**内容：三种"减点"手段（取边/面特征点、去地面点、下采样），**均无算法或公式**（没有 voxel-grid 滤波的体素中心/质心定义，没有 SOR 的统计判据，没有半径滤波阈值）。

**策略二的具体做法（源行 102）**——采用带高效近似的搜索结构，即便不总能给精确最近邻也能更快搜索。常见策略（全量保留）：
- 在**距离图像（range images）中做投影式邻居搜索（projective neighbor search）** [66]；
- 利用**体素网格（voxel grids）做近似邻居搜索** [1269, 256, 1133]。
- 此外，除欧氏空间纯几何对应搜索外，也可用**替代距离度量**、或**投影到特征空间（feature space）** [839] 来识别对应。

> 这是本源关于「最近邻 / KD-tree」的**全部**内容：① 精确 KNN 定义（式 8.3）+ "昂贵"动机；② 近似加速三法（range-image 投影搜索、voxel-grid 近似搜索、特征空间投影）。**KD-tree 本体的构建/查询/平衡算法在此并未出现**；KD-tree（GPU 加速、iKD-tree）要到 §8.2.2.3 才被点名，且仍只是"用途级"描述。

**图 8.4（LiDAR 里程计流水线各组件，源行 108）技术细节——核心系统图，全量保留**：给定一帧当前 scan：
1. **运动补偿（motion compensation）**：考虑 scan 过程中传感器运动，产生一帧去畸变（undistorted）scan；
2. **对应关系确定**：在去畸变 scan 与"过去的 scan（单帧或聚合在局部地图中的多帧）"间确定对应；
3. **相对位姿估计**：通过 scan registration 确定当前 scan 的相对位姿估计。
这些步骤**迭代**：对应集合基于中间相对位姿估计而被精化；收敛后最终位姿估计即里程计输出。

> **对点云处理章的提炼（综合可用）**：本源把 ICP 的"双步迭代"嵌进了里程计流水线——即 ① 去畸变（点云预处理）→ ②（重）建对应 → ③ 解变换 →（回到②）。这与点云配准章常讲的 ICP 循环一致；本源额外强调了"运动补偿"作为点云配准前的**预处理步骤**（详见下节）。

# 8.2.2 LiDAR 里程计常见组件（含点云预处理）（源 行 112–188）

本节关键模块：① 点云运动补偿；② 识别对应/对应搜索；③ 通过 scan registration 做位姿估计（scan-to-scan 或 scan-to-map）。

## 8.2.2.1 点云运动畸变补偿（点云预处理）（源 行 116–130；图 8.5）

> 说明：运动去畸变是 LiDAR 点云**特有的预处理**，点云处理章通常会涉及，故保留。

**问题来源（源行 118–122）**：运动畸变发生于 LiDAR 在一段连续运动期间采集一帧 scan。由于扫描期间移动，传感器在**略微不同的时间和位置**发射激光脉冲、接收测距返回。这意味着**单帧 scan 并非某单一位置在某单一时刻的静态快照，而是一组各自从略微不同扫描位置捕获的点**。若不校正，会导致不准确、产生畸变点云。
> **脚注 1（源行 120）**：现代 LiDAR SLAM 中 LiDAR 常装在移动车辆/机器人/可穿戴设备上；假设 scan 率 10 Hz，则 scan 周期 0.1 秒。
> **脚注 2（源行 136）**：相机图像捕获期间也有类似畸变，称**卷帘快门效应（rolling shutter effect）**。

去畸变（undistorting）是 LiDAR 里程计重要预处理步骤，可提升精度与鲁棒性。**四类补偿方法（源行 124–130），逐一全量保留**：

1. **恒速模型（Constant-Velocity Model）（源行 124）**：假设机器人保持与上一时间步估计相同的平移与旋转速度。不需任何额外传感器，可广泛用于较简单的 LiDAR 里程计系统 [256, 1133]；但运动含高频成分时，恒速假设本质上不够准确。

2. **连续时间轨迹优化（Continuous-Time Trajectory Optimization）（源行 126）**：用**样条（splines）[282, 715]** 与**高斯过程（GP）[50]**（参见 Chapter 2）的连续时间轨迹优化。连续时间轨迹允许在**任意时刻**做位姿估计而不依赖线性插值，可去除**每个单独点**的畸变；但常规连续时间轨迹优化耗时，常被离线实现 [715]。

3. **基于 IMU 的运动补偿（IMU-based Motion Compensation）（源行 128）**：IMU 用陀螺仪与加速度计直接测量高频运动，可在 scan 周期上积分作为有效补偿 [1269, 999]。做法：用最近 IMU 数据**预积分（pre-integrates）**出 LiDAR 位姿，再用该预测轨迹校正点畸变。得益于 IMU 高频率（如 200 Hz），对急剧（jerky）运动高度有效，现为大多数机器人平台事实标准。但须谨慎：IMU 测量噪声、偏置（bias）、估计误差、糟糕时钟同步都可能使其表现不如更简单方法。

4. **逐点配准（Point-wise Registration）（源行 130）**：最先由 **Point-LIO [442]** 提出。这种逐点方法与现有基于 scan 的框架**根本不同**：状态在**每个 LiDAR 点被接收时**就更新，而非累积一整帧 scan；因此该设计**不受 scan 内（intra-scan）运动畸变影响**。

**图 8.5（LiDAR 点云叠加在相机图像上，源行 146，From [1072]）技术细节**：(a) 无运动去畸变的 LiDAR-相机叠加；(b) 有去畸变的叠加。扫掠开始处（'b'）的轻微错位在扫掠结束时（'a'）显著恶化；底部——运动补偿后，开始（'d'）与结束（'c'）处都观察到更一致叠加。

## 8.2.2.2 基于特征的 LiDAR 里程计 = 点云特征提取（重点：LOAM）（源 行 132–158）

运动畸变校正后即可建立点对应。与视觉 SLAM（Chapter 7）类似，**利用特征做 scan 关联**已被充分研究。基于特征的方法允许**仅处理从点云提取的少量特征**来高效表示 scan；对应匹配与残差计算也可在特征层级进行，大幅降低整体计算成本。

### 低层特征（Low-level Features）（源 行 150–152）——LOAM 曲率 + PCA

- **线与面（Lines and planes）** 是实践中最常用特征。著名的 **LOAM 算法 [1269]** 是 LiDAR SLAM 的重大突破。LOAM 用一个**低层检测器**高效识别能贡献于 scan registration 的**中层特征**：
  - 通过**分析点与其邻居之差**计算**每个点的曲率（curvature）**；
  - **高曲率点 → 边特征（edge features）**；
  - **低曲率点 → 平面特征（planar features）**；
  - **并非所有点都用作特征**；算法选取**最显著的边点与面点子集**，以降低计算复杂度同时保持精度。

  > **综合 agent 注（LOAM 曲率公式缺失）**：源文只说"by analyzing the differences between the point and its neighbors"，**未给曲率的精确定义式**（LOAM 原文为 $c=\frac{1}{|\mathcal S|\cdot\|\mathbf X_{(k,i)}\|}\big\|\sum_{j\in\mathcal S,\,j\ne i}(\mathbf X_{(k,i)}-\mathbf X_{(k,j)})\big\|$），也未给边/面点的筛选阈值、每条扫描线分区取点上限、点-到-边线 / 点-到-面 残差及其雅可比。这些须从 `lidar_slam__loam_family.md` 或 LOAM 原论文获取。

- **主成分分析（Principal Component Analysis, PCA）** 也可用于识别点的**局部邻域主方向**以做有效特征检测 [67, 753]。通过计算"某点及其邻居的**协方差矩阵**"的**特征值与对应特征向量**，可确定主要变化轴，辨识点的几何性质：
  - **一个主导特征值**的点 → 分类为**边点（edge points）**，表示尖锐过渡或边界；
  - **两个相近特征值**的点 → 识别为来自**平面区域（planar regions）**，表示平坦表面；
  - 这使边特征与平面特征得以区分，通过考虑几何性质增强 LiDAR 里程计精度。

  > **综合 agent 注**：源文给出 PCA 判别边/面的**定性判据**（特征值的分布形态），但**未写出协方差矩阵 $C=\frac1N\sum(p_j-\bar p)(p_j-\bar p)^\top$、未写特征值 $\lambda_1\ge\lambda_2\ge\lambda_3$ 的线性度/平面度/散度指标的定义式**。

### 高层特征（High-level Features）（源 行 154–158）

如**语义、surfel、强度**特征，也能增强系统精度与鲁棒性，比低层特征提供更丰富的环境信息，利于更好的场景理解与更准确的建图。逐项全量保留：
- **语义特征（Semantic Features）**：用机器学习/深度学习把 LiDAR 点云**分类、标注**为对象类别（车辆、行人、建筑、植被）——通常用于**区分动态与静态对象**；可通过聚焦稳定地标提升里程计可靠性 [187]。
- **Surfel 特征（Surfels）**：surfel 是点云表面几何的**小圆盘状（disk-like）表示**（见 Chapter 5.3.2.2）。可对一组点拟合一个**椭球盘（ellipsoid disk）**，其**主半轴长度由邻近点协方差矩阵的特征值决定**。这种表面表示随后可用于在 scan registration 中**计算点-面距离** [846, 66, 903]。
  > 注：这里又一次出现"协方差矩阵特征值"——本源把 surfel 与 PCA 边/面判别共用同一思想，但仍**只给定性、不给公式**。
- **强度特征（Intensity Features）[277, 415]**：指 LiDAR 返回的**反射率 / 强度值**，提供环境中物体材料属性与表面特性的额外信息；通过提供额外信息维度提升特征匹配鲁棒性，在无结构（structure-less）等挑战场景尤为关键。

## 8.2.2.3 直接逐点 LiDAR 里程计（点 vs 特征 + KD-tree / iKD-tree）（源 行 160–166）

**基于特征方法的问题（源行 162）**：
- 倾向**丢弃那些不明确对应于面或边的孤立点的细微贡献**；在测绘带灌木/树枝的非结构化自然环境时尤为成问题；
- 在不同传感器间迁移时需调**手工设计的特征检测器（hand-engineered feature detectors）**；
- 后端要处理的点数通常随 LiDAR scan 大小**线性增长**；在现代 64 或 128 波束传感器下，计算成本会变得不切实际。

**直接法（源行 164–166）**：另一种做法是**直接用点**——不提取中层特征。与视觉 SLAM 直接法类似，可以 ICP 方式直接对齐点。但由于**逐点对应匹配的高计算成本**，在 LiDAR SLAM 早期不受青睐。
- **Zhou 等 [1295]**：为直接法铺路，用 **GPU 加速的 KD-tree** 实现加速最近邻搜索，使直接法变得实用。
- **Fast-LIO2（Xu 等 [1211]）**：之后的直接法，展示了高度准确的帧率（frame-rate）里程计，在地图增大时不受对应搜索瓶颈影响——靠其扩展即所谓**增量 KD-tree（incremental KD-tree, iKD-tree）**。**iKD-tree 能通过偶尔自我再平衡（rebalancing）来适应点的分布，实现高效的增、删、查（add/remove/query）操作，避免为每帧新增 scan 重建整棵树。**

> **综合 agent 注（KD-tree / iKD-tree 算法缺失）**：这是本源对 KD-tree 的**最深**叙述，但仍是"用途 + 一句特性"级别——**没有 KD-tree 的节点划分维度选择、构建递归、范围/最近邻查询、平衡条件，没有 iKD-tree 的惰性删除标记、子树重建触发判据、平衡准则**。这些（尤其 iKD-tree）须从 `lidar_slam__fastlio.md` 或 FAST-LIO2 原论文获取。

## 8.2.2.4 局部建图与位姿估计（点云局部地图 = 点云数据结构应用）（源 行 168–188；图 8.6）

获得可靠对应后，下一步是实现连续 scan 的一致配准。增量位姿估计大多通过某种 ICP 变体的 scan registration 实现（见 §8.2.1）。

- **scan-to-scan 里程计（源行 172）**：早期方法力求全频率（10 Hz）输出，把每次连续 scan 配准视为独立。但单帧 LiDAR scan 相对**稀疏**，若参考帧只是上一帧，许多点会有不合适对应，导致配准误差累积、整体地图不一致。
- **scan-to-map 里程计（源行 174–176）**：更现代——围绕传感器构建详尽准确的**局部地图（local map）**。已成功用于 2D LiDAR SLAM [458]、3D LiDAR 里程计 [1211, 256, 1133]、3D LiDAR SLAM [839, 66, 846]，并被观察到降低整体漂移率。机制：来自 scan-to-scan 或 IMU 预积分的运动预测，用于在对持久局部地图做精配准前**预对齐（pre-align）**进来的 scan；**局部地图比单帧 scan 稠密得多**，因而产生合适得多的**内点关联（inlier associations）**；配准后进来的 scan 被**加入此局部地图**（这正是 KD-tree/iKD-tree/体素地图作为"可增量更新的点云数据结构"的应用场景）。
- **单阶段 vs 双阶段历史（源行 178）**：较早方法 [1269] 因算力限制需**交错（interleaving）**——高频做 scan-to-scan、低频做 scan-to-local-map；更现代方法 [256, 1133] 用**单阶段 scan-to-map 对齐**，由**体素化局部地图表示**支持的**直接逐点对应**实现。
- **基于深度学习的里程计（源行 180–188）**：用深度学习直接从 LiDAR 测量学习自运动。早期用带真值标签的**监督学习** [655]；扩展到**无监督学习** [210]；性能总体有前景，但**泛化能力**受关切。

**图 8.6（结构感知混淆 structural perceptual aliasing，源行 186，来自 STheReO 数据集 [1262] 的 SNU Afternoon 序列）技术细节**：尽管 RGB 图像中 Place A 与 Place B 视觉差异很大（鸟瞰 vs 机器人前视），它们对应的 LiDAR scan 却呈现**结构上相似的模式**；这造成**结构感知混淆**——由于共享的道路拓扑与周围结构，不同地点在基于 LiDAR 的位置识别算法看来很相似。

# 8.2.3 LiDAR 里程计小结（源 行 190–192）

常见 3D LiDAR 里程计可通过**迭代地把进来的 scan 对齐到一个滚动的局部地图**产生高精度、鲁棒的运动估计——常借助 IMU 测量或运动模型校正 scan 运动畸变。系统可达 **每行进 1000 m 约 1 m 的漂移率**，但性能高度依赖机器人周围环境、场景动态程度、以及传感器自身动态。处理剩余的小漂移是 LiDAR SLAM 的关键，而位置识别是其关键组件。

# 与"点云配准"相关的其它散落要点（供综合补全）

> 这些点散落在 §8.3（位置识别）/§8.4（SLAM）/§8.5（趋势），与点云处理章的"配准/全局配准/不确定度"相关，择要保留。

- **全局配准（global registration，无初值的配准）（§8.4.1，行280）**：回环精配准需足够好的初值；小 pose-graph 可用几何先验，无几何先验时可用**现代全局配准方法 [1223, 665]**——不依赖初值即可鲁棒估计相对变换，**在两帧 scan 低重叠（low overlap）时仍工作良好**。（本源**未给全局配准算法**，如 RANSAC/TEASER/FGR 等，仅引用。）
- **几何验证（§8.4.1，行282）**：可用**旅行距离/时间差**、或 **RANSAC 对齐的置信度**做几何验证，判定回环候选有效性、避免加入错误回环。
- **ICP 用于回环精配准（§8.4.1，行278）**：位置识别只判定"两地相似"；确定精确相对变换需对相应 LiDAR scan 做**精配准（typically using ICP）**。
- **多任务点云对齐的"双墙（double walling）"问题（图 8.10，行316，From [951] ©2024 IEEE）**：(a) 朴素地直接对齐两个全局点云，**点-点距离**会出现"双墙"，幻化出虚假变化（phantom change）；(b) 多任务（multi-mission）位姿图松弛后，点-点距离减小、结构重建更清晰。（点云处理章讲"多次扫描融合的一致性"时可引。）
- **不确定度 / 协方差（§8.4.2 行310；§8.5 行368–370）**：可估计**数据驱动的协方差** [622,159] 把误差合理分配到 pose-graph（漂移率高处用高协方差边）；但源文坦言——**多数成功的 LiDAR 方法依赖 ICP，而 ICP 并不提供校准且鲁棒的位姿不确定度估计**（原文行368此句逻辑上像漏了"not"，应理解为"ICP 难给校准的不确定度"，故行370说估计的不确定度常不可靠、算法常用固定里程计协方差）。**本源不含 ICP 协方差 $\hat\Sigma=(J^\top J)^{-1}$ 类闭式估计的任何公式。**
- **鲁棒/无结构环境对点云配准的挑战（§8.5 行358–360）**：杂乱、无结构走廊、地下矿井、雪/雾/尘等极端天气会令 LiDAR SLAM 退化；最佳性能靠特征工程与人工调参（如 KISS-ICP [1133] 的自适应阈值），未来应按场景动态调整、引入 introspection。
- **多传感器融合（§8.5 行364）**：早融合（early fusion，单个紧耦合估计器）与晚融合（late fusion，分别单传感器估计再合并）；雷达在雨/烟中好、隧道中 LiDAR 失效可用视觉特征跟踪 [1191,1287]。
