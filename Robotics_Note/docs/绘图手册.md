| 模式 | 怎么做 | 字体自动统一? | 配只读 Docker? |

## 1. TikZ 与 tikz-3dplot

TikZ/PGF 是 LaTeX 下事实标准的矢量绘图系统：宏命令直接在 `.tex` 源码里描述图形，由 TeX 引擎排版成与正文同字体、同字号的图，无需外部图形软件。`tikz-3dplot`（Jeff Hein 著）是一个轻量扩展，提供“主坐标系/旋转坐标系”两套三维坐标变换样式与若干弧线/向量绘制命令，专门用于画三维坐标轴、向量、平面、角弧——非常契合 SLAM 中 SE(3) 位姿、旋转几何、对极几何这类示意图。下面所有代码均已在 **XeLaTeX** 下实测编译通过。

### 1.1 安装/环境

| 组件 | 随 TeX Live 发行情况 | 缺失时安装 |
|---|---|---|
| `tikz` / `pgf`（核心 + 键库 calc、positioning、arrows.meta、intersections、decorations.\*、fit、backgrounds、spy） | 含于 `texlive-pictures`（apt）或 `collection-pictures`（tlmgr）；完整版 TeX Live 默认已装 | `tlmgr install pgf`；apt: `sudo apt install texlive-pictures` |
| `standalone`（单图文档类，自动裁边，可被 `\includegraphics` 引入） | 含于 `texlive-latex-extra` | `tlmgr install standalone` |
| `tikz-3dplot` | CTAN 独立宏包；完整 TeX Live 通常已含 | `tlmgr install tikz-3dplot` |

**引擎**：本书用 `xelatex`（中文 + `fontspec` 友好）。TikZ/tikz-3dplot 对 `pdflatex`/`lualatex`/`xelatex` 均兼容；图含中文标注须用 `xelatex`/`lualatex`。**载入顺序**：`\usepackage{tikz-3dplot}` 必须写在 `\usepackage{tikz}` 之后。

> 本项目中这些库已由根目录 `figs.sty` 统一打包，正文 `\usepackage{figs}` 即用（preamble 已加载）。

### 1.2 核心语法/API

```latex
\usepackage{tikz}
\usepackage{tikz-3dplot}        % 必须在 tikz 之后
\usetikzlibrary{calc,positioning,arrows.meta,intersections,%
  decorations.pathreplacing,fit,backgrounds,spy}
```

常用键库职责：

- `calc` —— 坐标运算：`($(A)!0.5!(B)$)`（中点）、`($(A)+(1,0)$)`、`($(O)!1cm!(P)$)`（沿 OP 取 1cm）。
- `positioning` —— 相对放置：`node[right=2cm of A]{...}`、`below right=of B`。
- `arrows.meta` —— 现代箭头：`>={Stealth[length=3mm]}`、`-Latex`、`-{Stealth[round]}`（取代旧 `arrows` 的 `->`/`-stealth`）。
- `intersections` —— 求路径交点：路径加 `name path=p`，再 `\path[name intersections={of=p and q, by=X}];`。
- `decorations.pathreplacing` —— 最常用画大括号：`\draw[decorate,decoration={brace,amplitude=5pt}] (A)--(B);`。
- `fit` —— 用节点框住若干坐标：`\node[draw,fit=(A)(B)(C)]{};`。
- `backgrounds` —— `\begin{scope}[on background layer]...` 把网格/底色画到主图之下。
- `spy` —— 放大镜 inset（适合特征点匹配图的局部放大）。

**坐标系统**：直角 `(x,y)`（默认 cm）；极坐标 `(30:2)`（角 30°、半径 2）；三维 `(x,y,z)`；命名 `\coordinate (O) at (0,0,0);`。

**路径与节点**：

```latex
\draw[opt] (A) -- (B) -- cycle;          % 折线/闭合 (-- 直线; .. controls .. 贝塞尔; |- 直角折)
\filldraw[fill=blue!10,draw=blue] ...;   % 同时填充+描边
\node[style] (name) at (pos) {text};     % 节点; 锚点 anchor=north east 等
```

**`\foreach` 循环**（批量画刻度、采样）：

```latex
\foreach \i in {0,1,...,5} { \draw (\i,0) -- (\i,0.1); }            % 等差列表
\foreach \x/\lab in {0/O, 1/A, 2/B} { \node at (\x,0) {\lab}; }     % 多变量解包
```

**tikz-3dplot 核心 API**（命令名与参数顺序依官方手册）：

- `\tdplotsetmaincoords{θ}{φ}` —— 在 `tikzpicture` **之前**调用。**θ 绕 x 轴旋转、φ 绕 z 轴旋转**（度）。随后在环境或单条命令加样式 `tdplot_main_coords` 激活变换。常用经验值 `{70}{110}`～`{75}{120}` 立体感好。
- `tdplot_main_coords` / `tdplot_rotated_coords` / `tdplot_screen_coords` —— 主系 / 旋转系 / 还原成屏幕 2D（放图例、标题）。
- `\tdplotsetrotatedcoords{α}{β}{γ}` —— Euler 角 z(α)y(β)z(γ) 定义旋转子系，配 `\tdplotsetrotatedcoordsorigin{(P)}` 平移原点（画倾斜平面、机体系方便）。
- `\tdplotsetcoord{P}{r}{θ}{φ}` —— 由球坐标生成命名点 `(P)` 及其各坐标面投影点 `(Pxy)`/`(Pxz)`/`(Pyz)`/`(Px)`…（画“分量虚线盒子”一步到位）。
- `\tdplotdrawarc[坐标系,样式]{(中心)}{r}{起角}{止角}{标签选项}{标签}` —— 内置弧线（默认在主系 xy 面；空标签也要留 `{}`）。

### 1.3 关键用法模式

**模式 1：三维角弧的“版本无关”画法（推荐）。** `\tdplotdrawarc` 历史版本曾有比较 bug、且要求中心在 xy/x′y′ 平面、参数顺序易错。要在任意倾斜平面画弧或规避版本差异时，直接用 TikZ `plot` 参数式采样最稳：

```latex
% 主坐标系 xy 平面内, 0 到 \rotang 度画半径 0.5 角弧 + 箭头
\draw[->,thick]
  plot[domain=0:\rotang,samples=40,variable=\t]
    ({0.5*cos(\t)},{0.5*sin(\t)},0);
```

要画在倾斜平面内，把 `(x,y,0)` 换成该平面两正交单位向量的线性组合 `(r*cos t)*u + (r*sin t)*v`。本手册三例统一用此法。

**模式 2：分量分解盒子。** 用 `\tdplotsetcoord` 拿到 `(P)` 及投影点，虚线连 `(O)-(Pxy)-(P)`、`(Px)-(Pxy)` 即得标准“向量三轴投影”教学图。

**模式 3：屏幕层与图层分离。** 加不随视角变形的标题/图例用 `tdplot_screen_coords`；参考网格压到图下用 `backgrounds` 的 `on background layer`。

**模式 4：箭头/样式统一。** 在 `tikzpicture` 选项里设 `>={Stealth[length=3mm]}` 并定义 `vec/.style`，全图风格一致、改一处即全改。

### 1.4 完整案例

> 三例均为独立 `standalone` 文档，复制即可 `xelatex` 编译，输出自动裁边的单图 PDF。

**案例一：SE(3) 坐标三元组**

```latex
% Example 1: SE(3) coordinate triad
\documentclass[border=4pt]{standalone}
\usepackage{tikz}
\usepackage{tikz-3dplot}
\usetikzlibrary{arrows.meta}

% elevation 70 deg (rot about x), azimuth 120 deg (rot about z)
\tdplotsetmaincoords{70}{120}

\begin{document}
\begin{tikzpicture}[tdplot_main_coords,
    scale=2,
    >={Stealth[length=3mm]},
    axis/.style={thick},
    label/.style={font=\large}]

  \coordinate (O) at (0,0,0);

  % the three body axes of a frame {C}
  \draw[axis,red,->]   (O) -- (1,0,0) node[label,anchor=north east]{$\mathbf{x}_C$};
  \draw[axis,green!60!black,->] (O) -- (0,1,0) node[label,anchor=north west]{$\mathbf{y}_C$};
  \draw[axis,blue,->]  (O) -- (0,0,1) node[label,anchor=south]{$\mathbf{z}_C$};

  % a faint reference ground grid in the xy-plane
  \draw[gray!50,very thin] (-0.2,-0.2,0) grid[step=0.2] (1,1,0);

  \node[anchor=north east,fill=white,inner sep=1pt] at (O) {$\{C\}$};
\end{tikzpicture}
\end{document}
```

**案例二：罗德里格斯旋转的几何分解**（向量 $\mathbf v$ 绕单位轴 $\mathbf k$ 转 $\theta$：分解为 $\mathbf v_\parallel+\mathbf v_\perp$，转 $\mathbf v_\perp$ 再合成）。

```latex
% Example 2: Rodrigues rotation geometric decomposition
% v = v_parallel + v_perp ; rotate v_perp by theta ; recombine to v_rot
\documentclass[border=6pt]{standalone}
\usepackage{tikz}
\usepackage{tikz-3dplot}
\usetikzlibrary{arrows.meta,calc,decorations.pathreplacing}

\tdplotsetmaincoords{72}{118}

\def\vpar{1.2}   % length of v_parallel along k
\def\rperp{1.4}  % radius of rotation circle (|v_perp|)
\def\rotang{55}  % rotation angle theta in degrees

\begin{document}
\begin{tikzpicture}[tdplot_main_coords,
    scale=2.2,
    >={Stealth[length=2.6mm]},
    vec/.style={very thick,->},
    aux/.style={dashed,thin,gray}]

  \coordinate (O) at (0,0,0);
  \draw[vec,black] (O) -- (0,0,2.2) node[anchor=south]{$\mathbf{k}$};

  \coordinate (Vpar) at (0,0,\vpar);
  \draw[vec,blue] (O) -- (Vpar) node[midway,anchor=east]{$\mathbf{v}_\parallel$};

  \coordinate (Vperp) at (\rperp,0,0);
  \draw[vec,red] (O) -- (Vperp) node[midway,anchor=north]{$\mathbf{v}_\perp$};

  \coordinate (V) at (\rperp,0,\vpar);
  \draw[vec,black] (O) -- (V) node[anchor=south west]{$\mathbf{v}$};
  \draw[aux] (Vperp) -- (V);
  \draw[aux] (Vpar)  -- (V);

  \coordinate (VperpRot) at ({\rperp*cos(\rotang)},{\rperp*sin(\rotang)},0);
  \draw[vec,red!60!black] (O) -- (VperpRot)
        node[midway,anchor=south west]{$\mathbf{v}_\perp'$};

  \coordinate (Vrot) at ({\rperp*cos(\rotang)},{\rperp*sin(\rotang)},\vpar);
  \draw[vec,black!70!green] (O) -- (Vrot) node[anchor=south]{$\mathbf{v}_{\mathrm{rot}}$};
  \draw[aux] (VperpRot) -- (Vrot);
  \draw[aux] (Vpar)     -- (Vrot);

  % rotation circle traced by the perpendicular tip (version-proof parametric)
  \draw[thin,dashed,red!60!black]
        plot[domain=0:\rotang,samples=60,variable=\t]
          ({\rperp*cos(\t)},{\rperp*sin(\t)},0);

  % theta arc + label
  \draw[->,thick]
        plot[domain=0:\rotang,samples=40,variable=\t]
          ({0.5*cos(\t)},{0.5*sin(\t)},0);
  \node at ({0.66*cos(0.5*\rotang)},{0.66*sin(0.5*\rotang)},0) {$\theta$};
  \node[anchor=north east] at (O) {$O$};
\end{tikzpicture}
\end{document}
```

### 1.5 常见坑/最佳实践

1. **`\tdplotsetmaincoords` 必须在 `tikzpicture` 之外、之前**；写进环境内部图会退化成平面。
2. **θ 绕 x 轴、φ 绕 z 轴**（非“仰角/方位角”直觉命名）；θ 太小（近 0）会压扁成正视图。
3. **角弧优先用参数式 `plot`**，规避 `\tdplotdrawarc` 的版本/参数坑。
4. **三角函数单位是“度”**：`cos(30)`、`sin(\rotang)` 默认按角度；要弧度写 `cos(\t r)`。
5. **坐标表达式用花括号**：`({\rperp*cos(\t)},{\sin(\t)},0)`，否则逗号被误当分隔符。
6. **`arrows.meta` 取代旧 `arrows`**：用 `>={Stealth[...]}`/`-Latex`，勿混用过时 `-stealth`。
7. **旋转系内命名点限制**：`\tdplotsetcoord` 的预定义点只在主系有效；`tdplot_rotated_coords` 下用字面坐标。
8. **`scale` 安全，但勿手动覆盖 `x=`/`y=`/`z=` 基向量**（与 tikz-3dplot 投影冲突）。
9. **中文标注必须 XeLaTeX/LuaLaTeX** + `ctex`/`fontspec`。
10. **产物与引入**：`standalone` 输出紧贴图形单页 PDF，正文 `\includegraphics` 引入；`latexmk -xelatex` 自动多趟。

## 2. Asymptote（真三维数学图）

Asymptote 是一门描述性矢量绘图语言（语法接近 C++），随大多数 TeX Live / MiKTeX 附带（命令行 `asy`）。内置 `three`、`graph3` 模块，能用三元组 `(x,y,z)` 描述空间路径与曲面，并用 **LaTeX 排版所有标签**（公式 `$T_p\mathcal{M}$`、`$SE(3)$` 与正文同字体）。对 SLAM 中“流形与切空间、相机投影、坐标系三元组”这类**真三维**示意图，它是最佳工具，输出矢量 PDF。

> 本节代码均用 `asy`（TeX Live 2026 后端）实际编译通过。

### 2.1 安装/环境

- **随 TeX Live 自带**：`asy` 已在 PATH。验证 `asy --version`。
- 单独安装：`apt install asymptote` / `brew install asymptote`。`asy` 运行时仍需 LaTeX 工具链排版标签。
- **编译为 PDF**：
  ```bash
  asy -f pdf figure.asy             # 生成 figure.pdf（已自动裁剪到内容边界）
  asy -f pdf -render=0 figure.asy   # 纯矢量、无 OpenGL 预渲染（headless 必加）
  ```
  - `-render=0` 关闭 OpenGL 预渲染，得**纯矢量**；在无显示器的服务器/CI/容器中**必须加**，否则报 `freeglut: failed to open display`。**本项目 `build_figures.sh` 即用 `-render=0`。**
  - 也可 `-f eps`/`-f svg`/`-f png`。每个 `.asy` 默认生成单页 PDF，bounding box 紧贴图形（无需 `trim`/`clip`）。

### 2.2 核心语法/API

```asy
import three;            // 三维核心：path3/surface/triple、投影、光照
import graph3;           // 可选：三维坐标轴、参数曲线/曲面 graph()
size(9cm);
```

- **三元组与常量**：`triple p=(x,y,z);`；内置 `O,X,Y,Z`。向量运算 `cross(a,b)`、`dot(a,b)`、`unit(a)`、`length(a)`。
- **投影**：`currentprojection = perspective(camera=(4,2,3), up=Z, target=O);`（写实）或 `orthographic(...)`（无畸变，几何正确）。
- **光照**：`currentlight = White;`（着色）；`currentlight = nolight;`（**关闭光照**，干净线框/平涂——画线框球必用）。
- **画线**：`draw(path3 g, pen p, arrow);`，路径用 `--` 累积；参数曲线用循环累积；箭头 `Arrow3`/`BeginArrow3`/`Arrows3`。
- **圆/弧**（签名核对 `three.asy`）：`path3 circle(triple c, real r, triple normal=Z);`。纬线 `circle((0,0,z), r, Z)`；经线 `circle(O, R, (cos a,sin a,0))`。不确定签名时**参数化累积** `--` 自建最稳。
- **曲面**：`surface(path3 external)`；预定义 `unitsphere`/`unitdisk`/`unitplane`/`unitcube`。`draw(surface(quad), surfacepen)`。
- **点/透明/笔**：`dot(p, red+linewidth(3pt))`；颜色 `red/blue/heavygreen/gray(0.5)`；`dashed`/`dotted`；`opacity(0.3)`；字号 `defaultpen(fontsize(9pt))`。
- **LaTeX 标签**：`label("$T_p\mathcal{M}$", position, align);`（position 为 triple；align 用 N/S/E/W/NE…）。用 `\mathbf`/`pmatrix`/`\mathfrak` 等**须显式** `usepackage("amsmath"); usepackage("amssymb");`。

**嵌入 LaTeX**（推荐两步法：先 `asy` 生成 PDF，再 `\includegraphics`，与 XeLaTeX 主流程解耦）：

```latex
\includegraphics[width=.7\linewidth]{figures/pdf/so3_tangent.pdf}
```

### 2.3 关键用法模式

- **线框（无实体面）**：`currentlight=nolight;` + 只 `draw(circle(...))`/`draw(path3)`，**不要** `draw(surface(...))` 实心填充（避免遮挡）。
- **投影选择**：几何正确无畸变用 `orthographic`；写实纵深用 `perspective`。
- **配色约定**：坐标轴 $x/y/z$ → 红/绿/蓝；强调量用对比色 + 较粗 `linewidth`。
- **可复现**：固定 `asy -f pdf -render=0`。

### 2.4 完整案例

> ⚠ **数学正确性提示（务必在图注说明）**：把“流形 + 切平面”画成“**二维球面 $S^2$ + 切平面**”只是**教学卡通**。真实 $SO(3)$ 是**三维流形**（非二维球面），其切空间 $T_I SO(3)\cong\mathfrak{so}(3)$ 是 3 维。图注须注明“示意图(cartoon)”，勿让读者误以为 $SO(3)=S^2$。约定见 Solà 等《A micro Lie theory for state estimation in robotics》（arXiv:1812.01537）。

**案例 1：线框球面 + 切平面（流形与切空间，对应 $SO(3)$/$\mathfrak{so}(3)$）**

```asy
// Wireframe sphere + tangent plane: pedagogical cartoon of a manifold and
// its tangent space.  NOTE: only a cartoon -- SO(3) is a 3-manifold, not S^2.
// Compile: asy -f pdf -render=0 so3_tangent.asy
import three;
import graph3;
usepackage("amsmath");
usepackage("amssymb");           // for \mathfrak

size(10cm);
currentprojection = perspective(camera=(4,2,2.6), up=Z, target=O);
currentlight = nolight;          // clean wireframe: no shading
defaultpen(fontsize(9pt));

real R = 1;

// longitude circles (meridians): great circles with normal in XY
int nLon = 8;
for (int i = 0; i < nLon; ++i) {
  real phi = pi*i/nLon;
  triple nrm = (cos(phi), sin(phi), 0);
  draw(circle(O, R, nrm), gray(0.6)+linewidth(0.4pt));
}
// latitude circles (parallels): horizontal rings at fixed z
int nLat = 5;
for (int j = 1; j < nLat; ++j) {
  real z = -R + 2*R*j/nLat;
  real r = sqrt(R*R - z*z);
  draw(circle((0,0,z), r, Z), gray(0.6)+linewidth(0.4pt));
}
draw(circle(O, R, Z), gray(0.35)+linewidth(0.7pt));   // equator emphasised

// attach the tangent plane at a point p on the sphere
real a = 38*pi/180, b = 50*pi/180;
triple p = R*(cos(a)*cos(b), cos(a)*sin(b), sin(a));
triple n = unit(p);
triple u = unit(cross(n, Z));
triple v = unit(cross(n, u));

// tangent plane as OUTLINE + GRID (no opaque fill -> no occlusion)
real s = 0.85;
path3 quad = (p-s*u-s*v)--(p+s*u-s*v)--(p+s*u+s*v)--(p-s*u+s*v)--cycle;
draw(quad, blue+linewidth(0.6pt));
int ng = 4;
for (int k = 1; k < ng; ++k) {
  real tt = -s + 2*s*k/ng;
  draw((p+tt*u-s*v)--(p+tt*u+s*v), blue+linewidth(0.3pt));
  draw((p-s*u+tt*v)--(p+s*u+tt*v), blue+linewidth(0.3pt));
}

// point, a tangent vector xi, and the normal
dot(p, red+linewidth(3pt));
triple xi = 0.7*u + 0.45*v;
draw(p--(p+xi), red+linewidth(0.9pt), Arrow3);
draw(p--(p+0.6*n), heavygreen+linewidth(0.6pt), Arrow3);

label("$p$",            p,            1.4*W,  red);
label("$T_p\mathcal{M}$", p+s*u+0.9*s*v, NE,  blue);
label("$\xi$",          p+xi,         N,      red);

real ax = 1.6;
draw(O--ax*X, black+linewidth(0.5pt), Arrow3); label("$x$", ax*X, S);
draw(O--ax*Y, black+linewidth(0.5pt), Arrow3); label("$y$", ax*Y, S);
draw(O--ax*Z, black+linewidth(0.5pt), Arrow3); label("$z$", ax*Z, W);
```

**案例 2：相机投影（光心、像平面、3D 点、投影射线、像点）**

```asy
// Pinhole camera projection: optical centre C, image plane, world point P,
// projection ray, projected point p.  Compile: asy -f pdf -render=0 camera.asy
import three;
size(12cm);
currentprojection = perspective(camera=(5,-6,3.0), up=Z, target=(1.4,0,0.1));
currentlight = nolight;
defaultpen(fontsize(9pt));

triple C = O;                    // camera (optical) centre
real f = 1.5;                    // focal length (along optical axis +x)
triple pc = C + f*X;             // principal point
triple ex = Y, ey = Z;           // image-plane axes
real w = 1.1, h = 0.85;

// image plane as OUTLINE (no opaque fill)
path3 img = (pc-w*ex-h*ey)--(pc+w*ex-h*ey)--(pc+w*ex+h*ey)--(pc-w*ex+h*ey)--cycle;
draw(img, gray(0.45)+linewidth(0.7pt));
label("image plane", pc-w*ex+h*ey, NW, gray(0.3));

dot(C, black+linewidth(3.5pt));   label("$C$", C, W);
draw(C--(pc+1.1*X), gray(0.5)+dashed);                    // optical axis
dot(pc, gray(0.3)+linewidth(2.5pt)); label("$\mathbf{p}_c$", pc, SE, gray(0.3));

triple P = C + 3.4*X + 0.78*ex + 0.58*ey;
dot(P, blue+linewidth(3.5pt));   label("$P=(X,Y,Z)$", P, N, blue);

real t = f/(P.x - C.x);          // p = C + t (P-C) with x-coord == f
triple p = C + t*(P - C);
draw(C--P, blue+linewidth(0.8pt), Arrow3);
dot(p, red+linewidth(3.5pt));    label("$p$", p, SW, red);
draw(p--pc, red+dotted);

real al = 0.95;
draw(C--C+al*X, black+linewidth(0.6pt), Arrow3); label("$Z_c$", C+al*X, S);
draw(C--C+al*Y, black+linewidth(0.6pt), Arrow3); label("$X_c$", C+al*Y, S);
draw(C--C+al*Z, black+linewidth(0.6pt), Arrow3); label("$Y_c$", C+al*Z, W);
```

> 射线与像平面 `x=f` 交点用 `t = f/(P.x - C.x)`、`p = C + t(P-C)` 精确求得——针孔模型 $p \propto K[R\,|\,t]P$ 的几何版本。

**案例 3：$SE(3)$ 坐标系三元组**

```asy
// An SE(3) coordinate frame (right-handed triad) + world frame.
// Compile: asy -f pdf -render=0 se3_frame.asy
import three;
usepackage("amsmath");           // needed for pmatrix / \mathbf in labels
size(9cm);
currentprojection = perspective(camera=(5,4,3), up=Z, target=(0.6,0.4,0.3));
currentlight = White;
defaultpen(fontsize(10pt));

real L = 1.0;

void frame(triple o, triple ex, triple ey, triple ez, string sub) {
  draw(o--o+L*ex, red       +linewidth(1.1pt), Arrow3);
  draw(o--o+L*ey, heavygreen +linewidth(1.1pt), Arrow3);
  draw(o--o+L*ez, blue      +linewidth(1.1pt), Arrow3);
  label("$x_{"+sub+"}$", o+L*ex, align=SE, red);
  label("$y_{"+sub+"}$", o+L*ey, align=N,  heavygreen);
  label("$z_{"+sub+"}$", o+L*ez, align=W,  blue);
  dot(o, black+linewidth(3pt));
}

frame(O, X, Y, Z, "w");
label("$\{w\}$", O, align=SW);

triple t = (1.6, 1.1, 0.7);
transform3 Rt = rotate(50, O, (1,1,1));      // 50 deg about axis (1,1,1)
triple bx = Rt*X, by = Rt*Y, bz = Rt*Z;
frame(t, bx, by, bz, "b");
label("$\{b\}$", t, align=NE);

draw(O--t, gray(0.4)+dashed+linewidth(0.7pt), Arrow3);
label("$\mathbf{t}$", 0.5*t, align=N, gray(0.3));
label("$T_{wb}=\begin{pmatrix}R & \mathbf{t}\\ \mathbf{0} & 1\end{pmatrix}\in SE(3)$",
      (0.8, 0.2, 1.9), align=E);
```

> 旋转用内置 `transform3 rotate(real angle, triple O, triple axis)`（角度制），再 `Rt*X` 旋转基向量。`pmatrix`/`\mathbf` 依赖 `usepackage("amsmath")`。

### 2.5 常见坑/最佳实践

- **`pmatrix`/`\mathbf`/`\mathfrak` 报错或未定义**：asy 默认导言极简，**不**自动加载 `amsmath`/`amssymb`。用矩阵/`\mathbf` 加 `usepackage("amsmath")`，用 `\mathfrak`（如 $\mathfrak{so}(3)$）加 `usepackage("amssymb")`。（本项目 `so3_tangent.asy` 缺它即编译失败，加上即通过。）
- **透明/实心面遮挡**：`-render=0` 纯矢量模式下 `surface(...)` 的 `opacity(...)` 常**不生效**，半透明面渲染成不透明、挡住线框与标签。线框示意图**最稳是不画实心面**，只画外框 + 网格线。确需真半透明用 `asy -f pdf -render=4`（位图嵌入，非矢量）。
- **`freeglut: failed to open display`**（服务器/CI/Docker）：**加 `-render=0`** 强制纯矢量；或 `xvfb-run asy ...`。
- **光照让“数学图”变花**：几何示意图用 `currentlight = nolight;` 最干净；只有展示实体曲面质感才 `White`。
- **投影选择**：无畸变几何关系用 `orthographic`，写实纵深用 `perspective`。
- **标签压线/被挡**：用方位别名微调；仍重叠把锚点设为 `位置 + 偏移*基向量`。
- **签名不确定就参数化**：循环 `--` 累积自建圆/弧/面，最稳、可移植。
- **三轴配色一致**：全书统一 $x/y/z\to$ 红/绿/蓝。

### 3.2 核心语法/API

#### tikz-bayesnet 节点样式（“变量=圆、因子=小方块”的来源）

| 样式 | 形状/用途 |
|---|---|
| `latent` | 隐变量：白圆（`circle, fill=white, minimum size=20pt`） |
| `obs` | 观测变量：灰圆（`latent, fill=gray!25`） |
| `det` | 确定性节点：菱形（`latent, diamond`） |
| `const` | 常量/超参：无框文本 |
| `factor` | **因子：实心小黑方块**（`rectangle, fill=black, minimum size=5pt`） |

**宏签名**（`[...]` 可选，可留空 `{}`）：

```
\factor      [options] {name} {dir:caption} {inputs} {outputs}
\factoredge  [options] {inputs} {factors} {outputs}
\edge        [options] {inputs} {outputs}
\plate       [options] {name} {fitlist} {caption}
```

要点：
- **`\factor` 第 3 参=标签**，写 `方向:文本`（如 `{left:Multi}`、`{right:Dir}`），空写 `{}`。它内部建 `\node[factor,...]` 再调 `\factoredge`，并**自动生成 `<name>-caption` 标签节点**（做 `\plate` 时要把它列进 fitlist）。
- `\factoredge{inputs}{factors}{outputs}`：input→factor 画**无向**实线，factor→output 画**有向**箭头——正是因子图“变量—因子—变量”连法。先验因子写 `\factoredge {} {prior-f} {x1}`。
- `\edge{inputs}{outputs}`：变量间**有向**边（贝叶斯网）。`\edge[-]{a}{b}` 改无向。
- 多端：逗号列表 `\factoredge {x1,x2} {f} {l1}`。

> 本项目种子 `factor_graph.tex` 用更稳的写法：`\node[factor] (o1) at ($(x1)!0.5!(x2)$){};` + `\factoredge {x1}{o1}{x2};`（显式中点，避开不确定的定位键），已实测编译通过。

#### pgfgantt（核对官方手册 v5.0）

```
\begin{ganttchart}[<options>]{<start tss>}{<end tss>} ... \end{ganttchart}
```
- 两个必填参数=起止时间槽。常用键：`x unit`、`y unit title`、`y unit chart`、`hgrid`、`vgrid`、`today`、`bar/.append style`、`expand chart=\textwidth`。
- 元素：`\gantttitle{label}{slots}`、`\gantttitlelist{list}{len}`、`\ganttbar{label}{start}{end}`、`\ganttgroup{...}`、`\ganttmilestone{label}{tss}`、`\ganttlink{from-name}{to-name}`、`\ganttnewline`。

#### tikz-timing（核对官方手册 v0.7f）

逻辑字符（大写=整周期宽，小写=半宽）：`H` 高、`L` 低、`Z` 高阻、`X` 未定义、`D` 数据（`D{text}`）、`U` 未知数据、`T` 翻转、`C` 时钟、`G` 毛刺、`S` 空。

修饰：`nC`（宽度前缀，n 周期宽）、`{n}{chars}`（重复，如 `8{C}`、`3{LH}`）、`D{文本}`（总线标签）、`[<keys>]`（行内样式）、`;`（重置路径）。

表格环境：
```
\begin{tikztimingtable}[<settings>]
  <Signal Name> & [<row keys>]<chars>  \\
  ...
  \extracode \tablerules
\end{tikztimingtable}
```
缩放用 `timing/xunit`、`timing/yunit`、`timing/font`（**不要**用 TikZ `scale=`，它不正确缩放线宽/节点）。`timing/slope=0` 得理想方波。

#### decorations：花括号与路径标记

```latex
\draw[decorate, decoration={brace, amplitude=6pt, raise=4pt, mirror}]
     (A) -- (B) node[midway, above=10pt] {标注};
% 路径上等间隔箭头/刻度（postaction 叠加，不破坏原线）
\draw[postaction={decorate, decoration={markings,
      mark=between positions 0.1 and 0.9 step 0.1 with {\arrow{Stealth}}}}] (A)--(B);
```

**案例 1：SLAM 因子图（GTSAM/Dellaert 风格）**

```latex
\documentclass[border=6pt]{standalone}
\usepackage{tikz}
\usetikzlibrary{bayesnet}      % \factor \edge \factoredge \plate + latent/factor styles
\usetikzlibrary{positioning}
\begin{document}
\begin{tikzpicture}[x=2.2cm, y=2.0cm, every node/.style={font=\small}]
  % Variable nodes: poses on a row (latent = white circle)
  \node[latent]                 (x1) {$x_1$};
  \node[latent, right=of x1]    (x2) {$x_2$};
  \node[latent, right=of x2]    (x3) {$x_3$};
  % Landmark above the poses
  \node[latent, above=of x2]    (l1) {$l_1$};
  % Prior factor on x1 (no input, output = x1)
  \factor[left=of x1]    {fp}  {left:prior} {} {x1};
  % Odometry factors: pose -> pose
  \factor[above=of x1, xshift=1.1cm] {fo1} {above:$u_1$} {} {};
  \factor[above=of x2, xshift=1.1cm] {fo2} {above:$u_2$} {} {};
  \factoredge {x1} {fo1} {x2};
  \factoredge {x2} {fo2} {x3};
  % Measurement factors: pose -> landmark
  \factor[above=of x1, yshift=0.4cm] {fm1} {left:$z_1$}  {} {};
  \factor[above=of x3, yshift=0.4cm] {fm3} {right:$z_2$} {} {};
  \factoredge {x1} {fm1} {l1};
  \factoredge {x3} {fm3} {l1};
\end{tikzpicture}
\end{document}
```

**协方差/置信椭圆核心**：从 2×2 Σ 画椭圆，半轴沿 Σ 特征向量、半轴长 `sqrt(λ_i·s)`：
- 特征分解：`λ,V = np.linalg.eigh(Σ)`；半长轴 `a=sqrt(λ_max·s)`、`Ellipse(width=2a, height=2b)`；朝向 `angle=atan2(V[1,max],V[0,max])`。
- **卡方缩放（置信水平→s）**：2D 高斯马氏距离平方服从 χ²(2)。n-σ 椭圆 `s=n²`；p% 置信 `s=χ²₂⁻¹(p)`，**闭式 `s=-2·ln(1-p)`**（p=0.95→s=5.991，0.99→9.210）。注意 **1σ 椭圆在 2D 只覆盖约 39.3%**（非 1D 的 68%），图注务必写清“1σ”还是“95%”。

### 4.3 完整案例

**案例 1：2D 协方差椭圆（1σ/2σ/3σ + 95%）**

```python
# cov_ellipse.py  (numpy + matplotlib only)
import numpy as np
import matplotlib.pyplot as plt
from matplotlib.patches import Ellipse

def chi2_scale(conf):
    # chi-square (2 DOF) inverse CDF: P(X<=s)=1-exp(-s/2) => s=-2 ln(1-conf)
    return -2.0 * np.log(1.0 - conf)

def cov_ellipse(mean, cov, ax, n_sigma=None, conf=None, **kw):
    eigvals, eigvecs = np.linalg.eigh(cov)        # symmetric PSD -> real
    order = eigvals.argsort()[::-1]
    eigvals, eigvecs = eigvals[order], eigvecs[:, order]
    s = conf and chi2_scale(conf) or (n_sigma ** 2)
    width, height = 2.0 * np.sqrt(eigvals * s)    # full axis = 2 * semi-axis
    angle = np.degrees(np.arctan2(eigvecs[1, 0], eigvecs[0, 0]))
    e = Ellipse(xy=mean, width=width, height=height, angle=angle, fill=False, **kw)
    ax.add_patch(e); return e

rng = np.random.default_rng(2)
mean = np.array([3.0, 1.0])
cov = np.array([[1.2, 0.9], [0.9, 1.6]])
L = np.linalg.cholesky(cov)
pts = mean + (rng.standard_normal((400, 2)) @ L.T)

fig, ax = plt.subplots(figsize=(4.2, 4.0))
ax.scatter(pts[:, 0], pts[:, 1], s=6, alpha=0.3, color="0.5", label="samples", zorder=1)
ax.plot(*mean, "k+", markersize=12, markeredgewidth=2, label="mean", zorder=3)
for k, c in zip((1, 2, 3), ("C0", "C1", "C2")):
    cov_ellipse(mean, cov, ax, n_sigma=k, edgecolor=c, lw=1.6, label=fr"${k}\sigma$", zorder=2)
cov_ellipse(mean, cov, ax, conf=0.95, edgecolor="k", lw=1.4, ls="--", label="95%", zorder=2)
ax.set_aspect("equal")
ax.set_xlabel(r"$x$ [m]"); ax.set_ylabel(r"$y$ [m]")
ax.legend(loc="upper left", fontsize=8); ax.grid(True, ls=":", alpha=0.5)
fig.tight_layout()
fig.savefig("cov_ellipse.pdf", bbox_inches="tight", pad_inches=0.02)
```

**案例 2：NLLS 收敛曲线（cost vs 迭代，对数纵轴）**

```python
# nlls_convergence.py  (Gauss-Newton on y=a*exp(b*x))
import numpy as np
import matplotlib.pyplot as plt
rng = np.random.default_rng(0)
x = np.linspace(0, 2, 40)
y = 2.0 * np.exp(-1.3 * x) + 0.02 * rng.standard_normal(x.size)

def residual(p): a, b = p; return a * np.exp(b * x) - y
def jacobian(p):
    a, b = p; e = np.exp(b * x)
    J = np.empty((x.size, 2)); J[:, 0] = e; J[:, 1] = a * x * e; return J

p = np.array([1.0, -0.5]); costs = []
for _ in range(15):
    r = residual(p); costs.append(0.5 * r @ r)
    J = jacobian(p)
    dp = np.linalg.solve(J.T @ J, -J.T @ r)       # Gauss-Newton normal equations
    p = p + dp
costs = np.array(costs)                            # monotonically decreasing

fig, ax = plt.subplots(figsize=(4.2, 3.0))
ax.semilogy(np.arange(costs.size), costs, "o-", color="C0", lw=1.6, ms=5)
ax.set_xlabel("iteration"); ax.set_ylabel(r"cost $\frac{1}{2}\|r\|^2$")
ax.grid(True, which="both", ls=":", alpha=0.5)
fig.tight_layout()
fig.savefig("nlls_convergence.pdf", bbox_inches="tight", pad_inches=0.02)
```

### 4.4 常见坑/最佳实践

- **PDF（矢量）优于 PNG**：线条/文字任意缩放锐利。含百万散点/真实图像时可对该层 `rasterized=True` 或 `savefig(dpi=300)`，其余仍矢量。
- **`matplotlib.use("pgf")` 必须在 `import pyplot` 之前**。
- **`pgf.preamble`/`text.latex.preamble` 是字符串**（非列表）。
- **`eigh` 特征向量可能整体反号** → `angle` 差 180°，对椭圆几何等价无需修正。
- **1σ ≠ 68%（2D）**：1σ 椭圆覆盖约 39.3%；要 95% 用 `conf=0.95`（s≈5.99）。
- **协方差须对称半正定**：数值误差致非对称先 `cov=0.5*(cov+cov.T)`；用 `eigh` 比 `eig` 稳。
- **3D 等比例**：务必 `set_box_aspect((1,1,1))`，否则角度/长度“说谎”。
- **字号匹配正文**：`font.size` 设书稿 `\normalsize`，`width=...\linewidth` 让 PDF 1:1 嵌入（勿在 Python 出小图再被 LaTeX 放大）。
- **中文标签**：mathtext 无法显示中文；图内不放中文（用 `$x$`），或 pgf 后端 + `xeCJK` 同书稿字体。
- **可复现**：固定 `np.random.default_rng(seed)`，统一 rcParams。

## 5. 集成与工作流（SVG→PDF、GeoGebra、Manim、可复现管线）

> 本书在**只读源码目录**下用 XeLaTeX 于 Docker 内编译：编译期**不能**写中间文件，只取回最终 PDF。因此“编译时即时转图”（`svg` 包 `\includesvg`、即时 `epstopdf`）不可靠，**所有矢量图必须在只读编译之前预转成 PDF**。

### 5.1 格式真相

`\includegraphics`（pdf/xe/lualatex）只识别 **PDF / PNG / JPEG**，**不识别 SVG**；EPS 仅经 `epstopdf` 间接用（需可写环境即时转）。结论：**矢量交付格式唯一是 PDF**。所有 SVG/EPS/GeoGebra/Asymptote/脚本产物统一预转为 `.pdf` 再引用。

```latex

```

### 5.2 SVG→PDF 转换

**1) Inkscape 命令行（首选，矢量保真）。** 注意 1.x 与旧 0.x CLI 不同：

```bash
# Inkscape 1.x (current). --export-area-drawing crops to the actual ink.
inkscape figure.svg --export-type=pdf --export-filename=figure.pdf --export-area-drawing
#   legacy 0.x:  inkscape -z -D --export-pdf=figure.pdf figure.svg
```

**2) 字体匹配技巧 `--export-latex`（图中文字交给 LaTeX 排版）。** 把 SVG 拆成两份：

```bash
inkscape figure.svg --export-type=pdf --export-filename=figure.pdf --export-latex --export-area-drawing
#   figure.pdf      -> graphics only, NO text
#   figure.pdf_tex  -> text as real LaTeX (\put + \includegraphics{figure.pdf})
```

正文 `\input` 那个 `.pdf_tex`（不是 `\includegraphics`），用 `\svgwidth` 控尺寸：

```latex
\def\svgwidth{0.8\linewidth}
\input{figures/figure.pdf_tex}   % math like $\mathfrak{se}(3)$ now matches body font
```

**90/72 DPI 字号坑**：SVG 用户单位(px)与 pt 换算依赖 DPI——Inkscape 0.x 假定 90 px/in，1.x 改 96 px/in，而 LaTeX pt=72.27/in。`.pdf_tex` 里字号按 Inkscape 当时 DPI 写死；若 SVG 文档 DPI 与转换用的 Inkscape 版本不一致，文字会整体偏大/小。且 `\svgscale`/`\svgwidth` 只缩放**位置**不缩放**字号**（长期 bug）。对策：(a) Inkscape 文档 DPI 设为与主版本一致（1.x 用 96）；(b) 用 `\svgwidth` 而非 `\resizebox`；(c) 图不复杂时放弃 `--export-latex`，文字用 TikZ overlay 自己标注。

**3) 轻量替代（无 Inkscape，适合 CI/Docker）。** 对纯矢量 SVG 基本等价（但**不支持** `--export-latex` 文字拆分）：

```bash
rsvg-convert -f pdf -o figure.pdf figure.svg     # librsvg, tiny
cairosvg figure.svg -o figure.pdf                # pure Python
```

**4) 为什么只读编译里不用 `svg` 包的 `\includesvg`。** 它是**编译期调 Inkscape**：需 `--shell-escape` + Inkscape 在 PATH，且**必须向源目录树写**转换产物（默认子目录 `<jobname>-inkscape/`）。**本书源目录只读 → 这一步失败。** 正确做法是**编译前**用上面工具把 SVG 全部预转 PDF，编译期只 `\includegraphics`/`\input`。

### 5.5 可复现构建管线

核心原则：**版本库同时保留「可编辑文字源」（`.svg`/`.asy`/`.py`/导出的 `.tex`）与「构建产物 `.pdf`」**。一个 `build_figures.sh` 在**只读编译之前**把所有源重生成 PDF；只读阶段只消费 PDF。删掉所有 PDF 也能从源重建。

```bash
#!/usr/bin/env bash
# regenerate ALL figure PDFs BEFORE the read-only book compile.
set -euo pipefail
SRC=figures/src; OUT=figures/pdf; mkdir -p "$OUT"
for f in "$SRC"/*.svg; do [ -e "$f" ] || break; b=$(basename "${f%.svg}")
  if command -v inkscape >/dev/null 2>&1; then
    inkscape "$f" --export-type=pdf --export-filename="$OUT/$b.pdf" --export-area-drawing
  else rsvg-convert -f pdf -o "$OUT/$b.pdf" "$f"; fi
done
for f in "$SRC"/*.asy; do [ -e "$f" ] || break; b=$(basename "${f%.asy}")
  asy -f pdf -render=0 -o "$OUT/$b" "$f"; done
for f in "$SRC"/*.py;  do [ -e "$f" ] || break; ( cd "$OUT" && python3 "$f" ); done
for f in "$SRC"/*.tex; do [ -e "$f" ] || break; b=$(basename "${f%.tex}")
  latexmk -xelatex -interaction=nonstopmode -halt-on-error -output-directory="$OUT" "$f"
  rm -f "$OUT/$b".{aux,log,fls,fdb_latexmk}; done
```

> 本项目实际脚本见 `figures/build_figures.sh`（matplotlib 本地跑、asy/standalone-tex 在 texlive Docker 跑，因本机无 asy/xelatex）。

**权衡要诚实**：standalone/Asymptote/matplotlib 直接出的 PDF **不自动匹配正文字体**（文字按各自工具字体固化）。要字体一致只有：(a) SVG 走 `--export-latex` 的 `.pdf_tex`（代价是 DPI 字号坑）；(b) 产物 PDF 不含文字、文字用 TikZ overlay 由 XeLaTeX 排。关键公式标注图建议 (b) 或 `.pdf_tex`，纯几何/数据图接受工具自带字体即可。

## 7. 知识导航图标准（navtree / 分层流程图）

> 背景：早期各章「本章知识导航」用手工定位 TikZ（`below left=8mm and 2mm of ...` + `to[out=...]` 曲线边），
> 普遍**节点重叠**与**连线穿过文本框**。本节为强制标准：新图与重绘一律遵循，旧图按此整改。

### 7.1 先判形状：树 还是 DAG

- **树**（单根 → 支柱 → 叶，每节点恰一父，无跨支汇聚）→ 用 **`forest navtree`**（§7.2）。
- **DAG**（有多父汇聚节点，或跨支边承载核心语义、不可丢）→ 用 **分层 TikZ**（§7.3）。
- 判据：标出各节点入度，存在入度 ≥2 的节点 ⇒ 按 DAG 处理（forest 表不了多父）。

### 7.2 树形：forest `navtree`（首选）

样式已在 `figs.sty` 定义（`\RequirePackage{forest}` + `navtree`）：自动算层距/兄弟距（`l sep`/`s sep`），
兄弟子树**零重叠**，边走 `parent.south → child.north`**不穿框**，按层自动配色。用法（节点文字含逗号/方括号/数学/`\\` 一律 `{}` 包）：

```latex
\begin{center}
\begin{forest} navtree
[{根\\副标题}
  [{支柱A\\说明} [{叶}] ]
  [{支柱B} [{叶1}][{叶2}] ]
]
\end{forest}
\end{center}
```

- 配色按层自动（0/1/≥2 层 = main/second/third），勿在节点里手设 `fill`/`draw`。
- 勿写 `\small`（navtree 已设字号；裸 `\small` 有跨章泄漏风险，见易错手册）。

### 7.3 DAG 形：分层 TikZ（带汇聚 / 跨支边）

forest 只表树；DAG 用手工分层，但守**足距 + 正交布线**两规则，杜绝重叠/穿框：

- **定位**：按行（layer）排；同行列距用 `left=Xmm of`/`right=Xmm of`，X 取足够大使相邻框留白 ≥5mm。
  **禁** `below left/right=… and <≤3>mm`，**禁**负偏移（`and -Nmm`）把宽框往中间挤。
- **父子边**：竖直 `(a)--(b)`。
- **汇聚边**（多父→同子）：正交 `|-`/`-|` 绕到目标侧锚点，如 `(a) |- (sink.west)`、`(c) |- (sink.east)`；
  **禁** `to[out=..,in=..]` 曲线横跨（必穿框）。宽汇聚框用 `(parent) -- (parent |- sink.north)` 竖直入顶边。

范式（一根扇出三列 → 汇聚一框，正交不穿框）：

```latex
\begin{center}
\begin{tikzpicture}[>=Stealth,
  blk/.style={draw,rounded corners,align=center,font=\small,inner sep=3pt,fill=main!6,draw=main!60!black},
  ln/.style={->,thick,gray!60!black}]
\node[blk] (root) {根};
\node[blk, below=10mm of root] (mid) {中};
\node[blk, left=14mm of mid] (l) {左};
\node[blk, right=14mm of mid] (r) {右};
\node[blk, below=9mm of mid] (sink) {汇聚};
\draw[ln] (root)--(l); \draw[ln] (root)--(mid); \draw[ln] (root)--(r);
\draw[ln] (l) |- (sink.west); \draw[ln] (mid)--(sink); \draw[ln] (r) |- (sink.east);
\end{tikzpicture}
\end{center}
```

### 7.4 验收清单（每张导航图）

- [ ] 无任何两框重叠（含文字溢出压邻框）。
- [ ] 无连线穿过文本框（曲线、对角线尤其查）。
- [ ] 整图不超 `\textwidth`（超则精简节点文字/减列）。
- [ ] 树用 forest navtree；DAG 用 §7.3 正交分层。
- [ ] 无裸 `\small`（用 navtree 或 finenote 作用域）。
