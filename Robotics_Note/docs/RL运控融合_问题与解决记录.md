#### B.2 深层补遗（2026-06-24 第二轮：读 IPC/backend/algos 源码 + APPO 实跑；详见 `docs/RL运控融合_UniLab参考.md`）
> 本轮发现的 source-vs-认知 差异 + 易错点。深度参考全文在 `RL运控融合_UniLab参考.md`（架构内部 + 全编程模型 + 从零建任务 + 逐章 `\pz` 可落地代码）。
- **架构内部实跑确证**：APPO（`scripts/train_appo.py`，32 envs×3iter mujoco PASS）仪表盘直接显示 `Weight Sync 0.9ms / H2D Copy 1.4ms / Rollouts Read / Staging Pool 3/3 / Sync Collect ✓ / Vtrace Rho Clip` —— 即 `SharedWeightSync + RolloutRingBuffer + ReplayPipeline H2D + V-trace` 全在跑。IPC 全用 `mp.get_context("spawn")`（非 fork，避 CUDA 句柄复制崩溃）；collector 子进程崩溃经 `Pipe + ExceptionWrapper` 回传父进程，`format_collector_death` 把信号翻成排障提示（SIGBUS=7/OOM=9/SEGV=11）。
- **`SharedWeightSync`（ipc/weight_sync.py）**：learner→collector 权重同步用共享内存 `[float32 扁平 buffer | int64 version]`，`write_weights`(learner，version+1) / `read_weights_into`(collector，按 version 判断是否更新)——零序列化。这是异构设计核心，mjlab/Isaac 无对应（权重本在 GPU）。
- **off-policy 数据边界**：`ReplayBuffer(SharedBufferBase)` 是 **packed CPU 权威存储**（`torch.zeros(capacity, 2*obs+act+3+2*critic).share_memory_()`），CPU→GPU 经独立 `ReplayPipeline`（`CPUPinnedDoubleBufferReplayPipeline` 双缓冲 + pinned + 可选原生 `native_h2d_ext.cpp`，回退 `copy_(non_blocking=True)`；多卡 `MultiGPUCPUPinnedReplayPipeline` 每 rank 独立 H2D stream 不经 rank0）。`memory_budget.py` 在分配前算 `/dev/shm` 预算，超 80% 抛 `MemoryError`（共享内存特有失败面）。
- **后端 DR 能力非对称（精确）**：MuJoCo 后端 `supported_reset_terms` **11 项全集**（含 body_iquat/body_inertia/dof_armature）；Motrix **基线仅 4 项**（base_mass/base_com/body_mass/body_ipos）+ 能力门控追加 kp/kd/geom_friction/gravity，**不支持 body_iquat/body_inertia/dof_armature**。`DomainRandomizationManager` 经 `filter_reset_payload` 自动剔除不支持项（warning 跳过、不报错）。这解释 go2 motrix.yaml 显式 `randomize_kp/kd: false`。
- **reward dt 缩放确认**：`run_reward_dispatch` 返回 `reward * ctrl_dt`（等价 mjlab/Isaac 的 `scale_by_dt`，但乘 `ctrl_dt` 即控制周期）；每 `log_every_n_steps=4` 步写 `info["log"]["reward/<name>"]`（实跑日志确有这些键）。共享 reward 函数 ~29 个（`common/rewards.py`）。
- **非对称 actor-critic**：env `obs_groups_spec` 固定两组 `{"obs":<actor>,"critic":<critic>}`（go2 49/52、g1 98/101、manipulation 含 object_pos/quat + priv_info）；`split_obs_dict` 把 obs→actor、critic→critic。无独立命名 "privileged" 组——critic 组多带 base linvel 即特权。
- **HORA 蒸馏是真的**（`algos/torch/hora/distill.py` `HoraDistillationTrainer`）：stage-1 特权 PPO teacher（`mujoco_hora` owner）→ stage-2 student 只训 `adapt_tconv` 时序自适应编码器（teacher 主干冻结），RMA 范式。入口 `scripts/train_hora_distill.py`，回放 `play_interactive.py` 的 hora_distill 路径。**纠正初判**：UniLab 并非「无蒸馏」，而是「有 HORA 适应蒸馏、无通用 DAgger/视觉 student」。
- **ONNX 导出是真的且 onnxruntime 自验**（纠正「仅 play 阶段、疑似不完整」）：APPO/off-policy/HIM/MLX 都 `torch.onnx.export` 出 `policy.onnx` 并起 `ort.InferenceSession` 比对（打印 "ONNX export verified OK"）。时机差异：PPO 在 play/eval 阶段导（`training.no_play=true` 不导）；off-policy 由 `training.export_onnx`(默认 true) 控、训练内导。
- **sim2sim 跨后端契约校验**（`training/sim2sim.py`，GPU-sim 框架罕见）：`resolve_sim2sim_config` 比对源 run `run_config.json` 的 `contract_snapshot` 与目标 play cfg，DENYLIST 字段不一致时 `training.sim2sim_strict=true`(默认) 抛 `CrossBackendIncompatibleError`。motrix 在 tracking 任务「仅用于 sim2sim 评估/回放」注册。
- **motion_tracking 用 NPZ 参考运动**（`g1/motion_loader.py` `MotionLoader`，键 fps/joint_pos/body_pos_w/...），跟踪 reward `motion_body_{pos,ori,lin_vel,ang_vel}` + obs `motion_anchor_{pos,ori}_b`。`G1WBTObs` 是部署导向变体（去真机不可得通道）。
- **profiling = Perfetto/Chrome-trace JSON**（`logging/trace_event.py` `TraceRecorder`，`{"traceEvents":[...]}` → `perfetto_offpolicy_timeline.json`，`scripts/analyze_offpolicy_trace.py` 解析）。**无字面 `--profile` flag**（走 Hydra `training.trace_enabled`）；**trace 是 off-policy 路径特性，APPO 未接 TraceRecorder**。
- **NanGuard（utils/nan_guard.py）**：`capture(physics_state)` 环缓存 / `check(obs,reward)`+`check_ctrl(ctrl)` 返非有限 env ids / `dump` 写 `nan_dump_<ts>_step<N>.npz`(含 states 环缓冲 + meta + 复制 model_file + latest 软链)；`unilab-viz-nan` 用 `mujoco.viewer` 回放出事前帧。
- **注册名 vs CLI slug 两套**：注册名（`training.task_name`）CamelCase（`Go2JoystickFlat`，实测 **26** 个）；CLI `--task` 小写 slug（`go2_joystick_flat`）对应 `conf/<algo>/task/<slug>/<backend>.yaml`，经 owner YAML 的 `task_name` 绑定。
- **PPO 用 rsl-rl**（`from rsl_rl.algorithms import PPO`），自定义 `FinalObservationAwarePPO(PPO)`（处理终止观测 bootstrap，可 `torch.compile`）；APPO 基于 **IMPACT(Luo 2020)** V-trace + target net 软更新 + PPO clip；HORA 基于 RMA。
- **UniLab 真实欠缺（不编造，逐章如实标）**：① 视觉/相机策略任务（26 任务全本体感知）；② 对抗判别器 AMP/ASE 一等公民 + 通用 DAgger/视觉 student；③ `pseudo_inertia` 正定惯量 DR（mjlab 独有）；④ tracking_sigma「按步收紧」课程（有 PenaltyCurriculum 按 episode 长度缩放负权重 + TerrainCurriculum 地形等级）；⑤ learned actuator / 两级行星齿轮反射惯量；⑥ Manager 体系（非 Manager-Based，无 Obs/Reward/Event/Curriculum/Metrics/Recorder Manager 类）；⑦ 腱驱/位点力动作（动作=关节位置目标）。

### B.3 三框架架构精读：源码核到的事实与坑（2026-06-24）
- **`obs_groups` 键名两说并存**：Isaac Lab `RslRlOnPolicyRunnerCfg.obs_groups` 文档 docstring 示例用 **`{"policy":[...], "critic":[...]}`**（`rl_cfg.py:170`），但 locomotion 实配（及 B 节既载）用 **`{"actor":[...], "critic":[...]}`**。二者都是合法 key——具体哪个由**底层算法预定义的 obs-set 名**决定（rsl-rl 的 actor-critic 用 actor/critic；文档示例那段用 policy 指代 actor）。值永远是**env 提供的 obs-group 名列表**。复核时按"算法侧 set 名 → env 侧 group 名列表"理解，别纠结 policy vs actor。
- **Isaac Lab `step` 里 reset 发生在 reward/termination 之后、obs 之前**（`manager_based_rl_env.py:step`）：terminated/time_outs/reward 用的是**带 reset 前**的状态算，然后才 `_reset_idx`，最后 `observation_manager.compute()` 给出**reset 后**的 obs（注释明说"done after reset to get the correct observations for reset envs"）。这是标准 auto-reset 语义，写时序图务必把"先算奖惩→再 reset→最后取新 obs"画对。
- **mjlab `step` 的"单次 forward" 优化是其独有教学点**（`manager_based_rl_env.py:step` docstring，~60 行）：MuJoCo `mj_step` 内部先 forward-kinematics 再积分，故步后派生量(`xpos/xquat/site_xpos/cvel/sensordata`)滞后 `qpos/qvel` 一个 substep。mjlab **不**在 decimation 循环后 + reset 后各调一次 `forward()`，而是**只在 obs 计算前调一次** `sim.forward()`——一次刷新所有 env（非 reset env 拿到 post-decimation 运动学、reset env 拿到 post-reset 运动学）。代价：termination/reward manager 看到的派生量滞后一个 substep，但**滞后一致**（每 env 每步都一样）故 MDP 良定义。Isaac Lab 无此问题（PhysX 步后状态即时一致）——**这是 MuJoCo-warp 后端特有的设计取舍**，是 mjlab vs Isaac Lab 在 env.step 层最实在的差异之一。
- **mjlab 默认带一个 `reset_scene_to_default` event**：`ManagerBasedRlEnvCfg.events` 的 `default_factory` 就塞了它（mode=reset）。即"什么都不配 events 也会复位到初始态"；要彻底关闭须显式置空 events dict。Isaac Lab 无此默认（reset 行为靠用户在 EventCfg 里配 `reset_*` 项 + scene.reset）。
- **mjlab `auto_reset` 可关**（`ManagerBasedRlEnvCfg.auto_reset:bool=True`）：关掉后 `step()` 返回**真终止 obs**，调用方须自己 `reset(env_ids=...)`，否则下次 `step()` 抛 `RuntimeError`（`_manual_reset_pending` 守卫）。Isaac Lab 的 `ManagerBasedRLEnv.step` **无此开关**，永远 auto-reset（终止 env 在步内复位）。这是给自定义训练循环用的；mjlab 自带 train.py 走 rsl_rl OnPolicyRunner 不驱动手动 reset，故默认 True。
- **mjlab `scale_rewards_by_dt`（env cfg）↔ `RewardManager(scale_by_dt=)`**：env 把该开关透传给 RewardManager 构造器（实测签名 `RewardManager.__init__(self, cfg, env, *, scale_by_dt=True)`）。Isaac Lab 无对应 cfg 开关，奖励**总是** ×dt（`reward_manager.compute(dt)` 内 `value = func()*weight*dt`）。
- **任务注册机制两框架根本不同**（写 Ch01/Ch02 务必区分）：Isaac Lab 用**裸 `gym.register(id=..., entry_point="isaaclab.envs:ManagerBasedRLEnv", kwargs={"env_cfg_entry_point":"pkg.mod:Cfg", "rsl_rl_cfg_entry_point":..., "skrl_cfg_entry_point":"..yaml"})`**，cfg 以**字符串路径**存 kwargs，运行时 `load_cfg_from_registry()` 用 `importlib`+`getattr` 解析成类再实例化；任务发现靠 `import_packages()` 递归 walk 整个 `isaaclab_tasks` 包触发各 `__init__.py` 里的 `gym.register`。mjlab 用**自定义 `register_mjlab_task(task_id, env_cfg=<实例>, play_env_cfg=<实例>, rl_cfg=<实例>, runner_cls=...)`**（`mjlab/tasks/registry.py`），存的是**已实例化的 cfg 对象**到私有 dict `_REGISTRY`，`load_env_cfg()` 返回 `deepcopy`；任务发现靠 **Python entry_points（group=`mjlab.tasks`）** 在 `mjlab/__init__.py` 的 `_import_registered_packages()` 里 load——**不是** gym registry。故 mjlab 的 `list-envs` 读的是自己的 `_REGISTRY`，不是 `gym.envs.registry`。
- **配置系统两框架根本不同**：Isaac Lab 用自研装饰器 **`@configclass`**（`isaaclab/utils/configclass.py`，包 `dataclass` + 自动补类型注解 + 自动 `field(default_factory=)` 处理可变默认 + 注入 `to_dict/from_dict/replace/copy/validate`），配置写成**嵌套 `@configclass` 类**，term 是**类属性**（声明顺序 = obs 拼接/动作切片顺序，靠 Python 类体属性顺序保证）。mjlab 用**原生 `@dataclass(kw_only=True)`**（无自研装饰器），配置写成**工厂函数返回 dataclass 实例**，manager 配置是**普通 `dict[str, TermCfg]`**（如 `actor_terms={...}; observations={"actor":ObservationGroupCfg(terms=actor_terms,...)}`），顺序靠 dict 插入序。**这是"@configclass vs 原生 dataclass + dict"的核心对比，UniLab 第三说（Hydra dataclass + YAML）由 UniLab agent 覆盖。**
- **CPU↔GPU 边界两框架可见性不同**（数据流图关键）：mjlab 边界**显式可读**——`mujoco.MjModel/MjData`(host/NumPy) → `mjwarp.put_model/put_data(..., nworld=num_envs)` 抬到 warp 数组(GPU, `wp_device`) → `WarpBridge` 零拷贝暴露成 torch 张量（`sim/sim.py:_init_with_model/_finish_init`）。Isaac Lab 边界**藏在 Isaac Sim 内**——`sim.reset()` 那一下"activate physics handles that expose TensorAPIs"（`manager_based_env.py` 注释），之后 PhysX 状态以 torch 张量经 Fabric/TensorAPI 暴露，用户读不到 put_model 那层。教学上：mjlab 适合讲清"NumPy→warp→torch"全链，Isaac Lab 只能讲"reset 激活 TensorAPI 后一切皆 (num_envs,·) GPU 张量"。
- **mjlab CUDA-graph 是 4 张图**（`sim/sim.py:create_graph`）：`step_graph / forward_graph / reset_graph / sense_graph`，仅当 `_should_use_cuda_graph()`（warp mempool-enabled CUDA 设备）为真才 capture，`step()/forward()/reset()/sense()` 内 `wp.capture_launch(graph)` 重放、否则直调 `mjwarp.*`。**坑（已在 B.1 记 expand_model_fields）**：captured graph 锁死旧 GPU 数组地址，故 `expand_model_fields()` 换数组后**必须 `create_graph()` 重捕**——sim.py 在 `expand_model_fields`/`set_sensor_context` 末尾自动调 `create_graph()`，用户不手动碰。Isaac Lab 的图优化在 PhysX/Warp 内部，用户不可见也不需管。
