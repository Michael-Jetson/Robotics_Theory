## 2. 第 09 章 Radar SLAM

**覆盖度：完全没有（全新传感器）。**

**现有落点**：无专章。仅有零散"钩子"：`lidar_slam.tex:149`、`point_cloud_processing.tex:110/121` 提到 FMCW 测径向速度；`lidar_slam.tex` 多传感器融合处提"雨/烟雾用雷达"。RAE 观测模型（`sec:lidar-rae`）已声明"声呐、雷达等一束+两角传感器共享同一观测几何"——这是雷达接入的现成观测地基。

**Handbook 增量清单（几乎全是增量）**：
1. **雷达传感原理**：mmWave、RCS、旋转式雷达（radargram 极坐标图，>100m）vs SoC 片上雷达（datacube，3+1D/2+1D）；FMCW 测距（IF 中频，式 9.1–9.2）；锯齿调制的距离-多普勒耦合（式 9.3–9.5，`v=λΔφ/4πT`）；三角调制解耦距离/多普勒（式 9.6–9.9）；SoC 相位差测角（式 9.10）。**全套公式自包含，可全吸收。**
2. **雷达噪声**：斑点噪声、多路径"幽灵目标"、接收器饱和、运动畸变、低分辨率/高稀疏。
3. **雷达滤波**：CFAR / CA-CFAR / OS-CFAR / BFAR、k-strongest、统计阈值。
4. **雷达里程计四法**（核心增量）：(a) **多普勒里程计**——单扫描多点径向速度最小二乘解 ego-velocity，**无需对应关系**，配 IMU 补旋转（含因子图示例式 9.11–9.14，自包含）；(b) 直接法（相位相关/Fourier-Mellin/Masking-by-Moving CNN）；(c) 特征法（FSCD/BASD 雷达专用描述子）；(d) 配准法（CFEAR 多关键帧+法向/平面性加权、APDGICP）。
5. **雷达位置识别**：学习法（圆柱卷积旋转不变+NetVLAD）、描述子适配（ScanContext 用 RCS/强度代高度、RING、M2DP 的 2D 适配）。
6. **雷达 SLAM 系统**：TBV-SLAM、RadarSLAM、4DRadarSLAM、4D iRIOM；地图表示（landmark/occupancy/heatmap/NDT/神经场/PHD 滤波动态栅格）。
7. **多模态**：雷达-LiDAR（能见度条件信任）、雷达-相机、雷达-IMU、雷达-热像、先验地图（OSM/卫星图 RSL-Net）。
8. **数据集**：Oxford Radar RobotCar、MulRan、Boreas（全天候）、nuScenes、ColoRadar、K-Radar 等（表 9.1，11+）。

**融合策略（重中之重）**：**新建独立一章 `ch:radar`，但用"对照 LiDAR/IMU 已学内容"的方式织入主线，避免孤岛。** 雷达与激光同属 range 传感、与 IMU 同走预积分/因子图，可大量**复用已建立的语言**，这是让它"读起来连贯"的关键：
- **观测地基复用**：开篇承接 `sec:lidar-rae` 的 RAE 几何——"上一章我们把激光的一束+两角读作 RAE 观测；雷达虽用电磁波，其极坐标 radargram 同属这一观测族，差别在它额外免费给出一维**径向速度**。"由此把多普勒作为雷达相对激光的"独有维度"引出，主线立刻接上。

## 3. 第 10 章 Event-based SLAM

**覆盖度：完全没有（全新传感器）。**

**现有落点**：无专章。钩子：`intro_slam.tex:217` 已提"事件相机等尚未成主流的特殊种类"；`imu_model.tex:1109/1117` 在连续/GP 预积分处两次提到"对事件相机等非硬件同步传感器尤有用"——这是事件-惯性融合的现成接口。

**Handbook 增量清单（几乎全是增量）**：
1. **事件相机原理**：事件四元组 `e=(x,y,t,p)`；事件生成模型（式 10.1：对数亮度变化达阈值 `ΔL=p·C`，自包含）；异步、像素独立；优势（HDR>120dB、低延迟亚毫秒、无运动模糊、高时间分辨率、低功耗、数据驱动）；生物视网膜启发；DVS/DAVIS/ATIS 器件演进。
2. **事件表示**：事件帧、时间面/SAE（Surface of Active Events）、体素网格、运动补偿事件图（IWE）。
3. **方法分类**：逐事件 vs 批处理；模型 vs 学习（ANN/SNN）；间接（特征：点/线/法向光流）vs 直接（全事件）；几何 vs 光度目标。
4. **直接法核心**：**对比度最大化 / 聚焦框架（Contrast Maximization, CMax）**——通过最大化 IWE 的方差/梯度估计运动（自包含概念，是事件 SLAM 的标志性方法）。
5. **后端**：间接（重投影误差）/ 直接（光度或事件对齐最大化）；多数事件 SLAM 缺后端，少数用滑窗 BA。
6. **代表系统**（表 10.1，39 个）：EVO、EKLT、ESVO/ESVO2、Ultimate-SLAM(USLAM)、EDS、CMax-SLAM、DEVO/DEIO、ESVIO。
7. **事件-帧/事件-惯性融合**、旋转估计（全景 SLAM）、运动分割（IMO）、图像重建（Poisson 积分）。
8. **数据集/模拟器**（表 10.2，17 个 + ESIM/V2E/Vid2E/V2CE/EventGAN）；评估指标（ATE/RPE/ARPE/ARRE/AEE）。

**融合策略（重中之重）**：**最优是与雷达/腿式合成一章/一节群（见 §6），或新建独立短章 `ch:event`；织入靠"对照帧相机 VO 与连续时间估计"。**
- **承接视觉里程计与相机模型**：事件相机是 `ch:camera`/`ch:vo` 的"异步孪生"。开篇对照"普通相机同步输出整帧强度、事件相机异步输出亮度变化"，把 HDR/低延迟/无运动模糊作为"补帧相机之短"引出。CMax 与直接法 VO（`ch:vo` 的直接法/光度误差）作平衡对照。
