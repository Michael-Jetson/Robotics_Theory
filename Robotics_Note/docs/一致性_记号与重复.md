## ① 记号不一致清单

### 1.1【严重】SE₂(3) 群元素：5×5 矩阵列序相反 + 位置字母不一致
- **`parts/P0_math/lie_theory.tex:986`**（权威定义 `sec:se23` / `eq:se23-group`）：
  `T = [[R, r, v], [0,1,0], [0,0,1]]`，即 **第 2 列 = 位置 r，第 3 列 = 速度 v**；李代数 `ξ=[ρ;ν;φ]`（位置;速度;旋转），指数映射同序 `[[R, Jρ, Jν], …]`（:1017），伴随行序也是先位置块 `r∧R` 后速度块 `v∧R`（:1038）——**内部完全自洽**。
- **`parts/P1_estimation/kalman_eskf.tex:979`**（InEKF 紧凑版 `paper:inekf`）：
  `X = [[R, v, p], [0,1,0], [0,0,1]]`，即 **第 2 列 = 速度 v，第 3 列 = 位置 p**；且 :982 文字"含 R/v/p"再次强化"速度在前、位置在后"。
- **冲突点**：同一个群元素，两处把**速度列与位置列放在交换的位置**，且位置变量一处记 `r`、一处记 `p`。`kalman_eskf.tex:982` 明确 punt"正式定义见 `\cref{sec:se23}`"，但读者跟过去会撞上列序/字母不一致。`imu_model.tex:1114`、`vio.tex:1147` 也都回指 `sec:se23`+`paper:inekf`——**3 章共享这两个口径不一的源**，影响放大。
- **建议**：以 `sec:se23` 为唯一权威。`kalman_eskf.tex:979` 改为 `[[R, r, v], …]` 列序、位置改记 `r`（与 :986 对齐）；或反之统一为 InEKF 文献惯例 `[[R, v, p], …]` 并回改 `sec:se23` 全套（含 :991/:1017/:1038/:1014 的分量序）——**二选一，但 `sec:se23` 是被回指的权威，改 kalman 一处代价最小**。

（次要）群元素记号：`sec:se23` 用 `T`、`paper:inekf` 用 `X`。X 是 InEKF 文献惯例（记群状态），可接受，但若顺手亦可统一。

### 1.2【中】`nonlinear_optimization.tex` 内 `\mathbf{P}` 三义同篇

- 抽取/膨胀矩阵：:1255–1262、:1278、:1600（符号表）`\mathbf{P}_k`（ESGVI 因子变量抽取）。
- **建议**：ESGVI 抽取矩阵 `\mathbf{P}_k`（:1255 起）换字母（如 `\boldsymbol{\Pi}_k`），并在符号表 :1600 显式区分"协方差 P / 三维点 P' / 抽取矩阵"。其余两义是全书既有约定（卡尔曼 P=协方差、notation P=点），保留即可。

### 1.3【中】连续时间 GP：白噪声 PSD 与连续系统矩阵字母漂移
集中在 `nonlinear_optimization.tex` 附录那段向量空间 GP derivation（约 :1843–1844），与 STEAM / Van Loan 的口径不一致：

- **连续系统矩阵**：附录 :1844 用 `\mathbf{F}(t)`，而 STEAM :1227 与 Van Loan :582 用 **`\mathbf{A}`**；且 `matrix_calculus.tex:737` 处 `\mathbf{F}=∂f/∂x` 又是**离散运动雅可比**之义——附录拿 F 表连续系统矩阵会与 A 及 KF 章离散 F 双重撞名。**建议附录 :1844 改 `\mathbf{A}(t)`**。

### 1.4【低】`slam_state_estimation.tex`：`\mathbf{v}` 输入 vs 观测噪声双关

### 1.5【低】`slam_state_estimation.tex` STEAM 协方差用 `\mathbf{P}` 但章首未声明

## ② 重复 / 矛盾清单

### 2.1【一致，无冲突】SE₂(3) 定义之外的项
- ξ 分量顺序：两处方向一致（平移类在前、旋转在后），`paper:inekf` 未显式列 9 维顺序，已 punt 到 `sec:se23`。
- 伴随 Ad(T)：仅 `lie_theory.tex:1038` 给出 9×9，`paper:inekf` 未列、已 punt。无对比冲突。
- 左/右扰动：InEKF 用**右不变误差** `η=X̂X⁻¹`（`kalman_eskf.tex:973`）+ **右扰动** `X·Exp(δξ)`（:982），与全书右扰动主线**一致**，且 :970/:984 多处显式声明"同样右扰动""与本书右扰动主线天然契合"——不存在"左不变 vs 右扰动主线"矛盾。
  （唯一实质矛盾是 §1.1 的矩阵列序/字母，已在记号清单列出。）

### 2.2【重复+缺交叉引用】鲁棒核全家族：kalman 与 nlopt 平行两份
两文件对**同一组主题（鲁棒核 + IRLS + RANSAC + 逆 Wishart MAP）各完整写一遍**，数学结论自洽，但**彼此无任何针对性 `\cref` 互指**（只有泛泛的 `\cref{ch:nlopt}`/`\cref{ch:eskf}` 互指 GN/EKF 关系）。读者会疑惑"同一个东西写了两遍"。

- **鲁棒核 ρ 函数**
  - `kalman_eskf.tex:1050–1054`（`eq:robust-kernels`）：Huber/Cauchy/GM 三个。
  - `nonlinear_optimization.tex:961–975`（表 `tab:robust`）+ :979–981：L2/Huber/Cauchy/GM/**Tukey** 五个。
  - **Cauchy 公式不一致**：`kalman_eskf.tex:1052` 写 `½ln(1+u²)`（**无尺度 c**），`nonlinear_optimization.tex:970` 写 `(c²/2)ln(1+u²/c²)`（**带尺度 c**）。Huber、GM 两处一致。
  - **Tukey 仅在 nlopt**（:972/:979–981，`c=4.685`），kalman 无——非冲突，但说明两份"全家福"不对称（kalman 是 nlopt 子集）。
  - **建议**：以 `tab:robust`（nlopt，带 c）为主讲。`eq:robust-kernels` 缩为一句"二范数换增长更慢的 ρ"+ `\cref{tab:robust}`；或在 kalman 处注明"取 c=1 的简化"。Tukey 不必在 kalman 重复。

- **RANSAC（两处几乎逐字重复、公式字面相同）**
  - `kalman_eskf.tex:1041–1046`（`eq:eskf-ransac`，`k=ln(1−p)/ln(1−wⁿ)`）。
  - `nonlinear_optimization.tex:945–951`（`eq:nlopt-ransac`，同一公式）。
  - 习题也各重复同一道（`w=0.5,n=2,p=0.99`）：`kalman_eskf.tex:1088` 与 `nonlinear_optimization.tex:1045`。
  - **建议**：主讲留 nlopt（其 Tukey 段 :980 已引 `eq:nlopt-ransac`，且与"软降权 vs 硬剔除"对照最自然）。kalman 处删五步细节与公式，改 `\cref{eq:nlopt-ransac}`；删重复习题之一。

- **逆 Wishart（同一推导两遍、无互指）**
  - `kalman_eskf.tex:1064–1077`（`thm:robust-iw`/`eq:iw-cov`，要点级）。
  - `nonlinear_optimization.tex:1008–1034` + 附录 :1732–1734（`eq:nlopt-invwishart*`，完整级 + Sherman–Morrison 全步）。
  - 结论一致（Ψ=Σ 时 = 加权 Cauchy），但**重复且无 `\cref` 互指**。
  - **建议**：kalman 的 `thm:robust-iw` 保留"结论+一句动机"，推导改 `详见 \cref{ch:nlopt}`（nlopt 完整证明）。

- **IRLS 膨胀协方差（同一推导两遍 + 字母不一致）**
  - `kalman_eskf.tex:1057–1062`（`eq:irls-weight`，膨胀协方差记 **`\mathbf{Y}_i`**；Cauchy 例 :1062）。
  - `nonlinear_optimization.tex:993–1003`（`eq:irls`、`eq:nlopt-cauchy-Y`）。
  - **字母不一致**：同一"IRLS 膨胀协方差"kalman 记 `\mathbf{Y}_i`、nlopt 侧记法不同（与逆 Wishart 最优协方差 `\mathbf{M}_i` 同族对象在两章字母不统一）。
  - **建议**：统一膨胀协方差/权矩阵字母（倾向跟 nlopt），并加跨文件 `\cref`，避免读者以为是两个不同结果。

### 2.3【一致，已正确复用】立体相机贯穿例数值
- 物理参数 `kalman_eskf.tex:454` 设定（`b=0.1m, f=400px, R=0.09px², x̌=20m, P̌=9m²`），`nonlinear_optimization.tex` 明确声明"参数同 `\cref{eq:stereo-bayes}`/`\cref{sec:ekf}`"——**刻意复用、未重设**。

### 2.4【互补三层，缺一条反向引用】连续时间 GP 三处推导
三处推导同一族 LTI SDE→GP/离散协方差，但**分工互补、非失控冗余**：
- `nonlinear_optimization.tex` 附录（约 :1843–1844）= **向量空间骨架**（LTV 一般情形）。
- `slam_state_estimation.tex:1181+` = **SE(3) 局部 GP 落地**（WNOA、A 幂零手算闭式）。
- `matrix_calculus.tex:545+` = **离散化引擎**（Van Loan，通用算法）。
- **交叉引用大体完整**：`slam_state_estimation.tex:1188/1335` → nlopt 附录；`matrix_calculus.tex:548/628/732/737` → `sec:est-steam`。
- **唯一缺口**：STEAM 的闭式 Φ,Q 推导（`slam_state_estimation.tex:1227–1239`，`eq:steam-Phi-Q`）**未反向引用 Van Loan 定理**（`matrix_calculus.tex` 的 `thm:matderiv-vanloan`）。是 Van Loan 那侧单向引了 STEAM。
- **建议**：`slam_state_estimation.tex:1239` 或 :1384（"换 SDE 只换 A,L,Φ,Q"处）加一句 `\cref{thm:matderiv-vanloan}`，三处闭环（"幂零可手算 / 一般用 Van Loan"显式接上）。符号统一见 §1.3。
